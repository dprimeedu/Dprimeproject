# 단어훈련(Quizlet식) 신규 기능 — 작업 핸드오프

> 다른 프로젝트(문서자동화 `완성(코드)`)에서 내신단어 데이터 파이프라인을 만든 뒤, 그 단어를 이 홈페이지에서 훈련시키기 위해 넘어온 작업. 이 md 하나로 시작 가능.

## 목표
재원생 메뉴(`/training/`)를 **카드 2개**로 재구성하고 **단어훈련**을 신규 추가한다.
- 재원생 메뉴 = **영작훈련** + **단어훈련**
- 기존 **대전입장**은 재원생 메뉴 top에서 빼고 **영작훈련 안**(writing 홈)으로 이동
- 단어훈련 = **Quizlet식**, 기존 `writing`(영작훈련) 앱 구조를 미러로 제작

## 현재 구조 (확인됨)
- 재원생 메뉴 페이지: `config/templates/training.html` (현재 카드: 영작훈련 + 대전입장)
- 사이드바: `config/templates/base.html` 86~94줄 (재원생 → 영작훈련 / 대전 입장)
- 영작훈련 앱: `writing/` — Quizlet식 완성형. 단원→문제→배정→세션→단어시도 + XP/레벨/배지/대전/라이브모니터
  - 미러 대상 모델(`writing/models.py`): `WritingUnit`(단원)·`WritingProblem`(한글→영어)·`UnitAssignment`·`WritingSession`/`WritingAttempt`·`StudentProfile`(XP/레벨)·`StudentUnitLevel`·`MatchRoom`/`MatchParticipant`(대전)
  - 라우트(`writing/urls.py`): 학생 `/training/`(home, unit start, flashcard, session, result) + admin(엑셀 업로드, 단원/학생 관리, 대전)
  - 엑셀 업로드 형식: `writing/services/excel.py` `parse_writing_excel` → [색인, 영어, 한글] 3열
- 학생(유저) 모델: `member.Member` (`login_id`=학원 재원생 ID, `username`=이름, `nickname`, `is_approved`=재원생 승인). 재원생 메뉴는 승인 필요.

## 단어훈련 설계 (신규 `vocab` 앱 = writing 미러)
| writing | → vocab |
|---|---|
| WritingUnit(단원) | **VocabUnit** (학교/시험 단위, 예: 동백고1 1학기 기말) |
| WritingProblem | **VocabWord** (단어·뜻·단원·출처) |
| UnitAssignment | **VocabAssignment** |
| WritingSession/Attempt | **VocabSession/VocabAttempt** |
| StudentProfile(XP) | **재사용** (영작/단어 XP 통합) |
| — | **StudentWordStar** (학생×단어 별표 = 모르는 단어) |

UI: writing flashcard/session 복제 — 단어↔뜻 플래시카드 + 별표 단어 집중훈련(Quizlet "어려운 단어" 식).

## 단어 데이터 출처 (★다른 프로젝트에서 생성됨, import로 가져옴)
- **통합본 내신단어 xlsx**: 교재폴더의 `{학교} {시험} 통합.xlsx` → `내신단어` 시트
  = `[색인, 단어, 해석, 단원, 출처]`
  - 예: `Z:\home\Drive\교재폴더\내신\고등학교 1학년\개정판\2026년\2026년 1학기 기말고사\동백고1\동백고1 1학기 기말고사 통합.xlsx`
  - (학년별 base: 고1 `고등학교 1학년\개정판\2026년`, 고2 `고등학교 2학년\개정본\2026년`, 고3 `고등학교 3학년`)
- **학생 별표(모르는 단어)**: 학생 구글시트 "내신단어" 탭 = `[색인, 단어, 해석, 체크(D), 단원, 출처]`
  - **D열 '체크' = 학생이 표시한 '모르는 단어'** → StudentWordStar 초기값으로 이식 (별표)
- import 방법: `vocab` 앱에 **Django management command** 작성해 위 xlsx(`내신단어` 시트) 읽어 VocabUnit+VocabWord 생성. (홈페이지 venv 에서 `python manage.py import_vocab ...`)
  - openpyxl 사용 (requirements 에 이미 있을 것, writing/services/excel.py 참고)

## 진행 순서
1. **nav 재구성 + 대전 이동** — `training.html`(영작훈련+단어훈련 2카드), `base.html` 사이드바, writing 홈에 대전입장 폼 이식 (DB 무관, 안전)
2. **vocab 앱 + 모델 + 마이그레이션** (추가 테이블만)
3. **import management command** (통합본 xlsx → VocabUnit/VocabWord)
4. **플래시카드/퀴즈 UI** (writing 미러) → 별표/게임화/대전 순 확장

## 주의
- **라이브 사이트** + `db.sqlite3` 약 64MB. 마이그레이션은 추가 테이블만이라 안전하지만 **적용 전 확인**.
- 이 프로젝트의 `CLAUDE.md`/규칙·`.claude` 설정·venv(`project_env`, requirements.txt) 먼저 확인하고 시작.
- 학생 매핑: 내신단어 학생명 ↔ `member.Member.username`(이름) 기준. (학원 재원생은 login_id 보유)

## 큰 그림: 구글시트/Apps Script → Django 이전 (단어훈련이 1단계)

현재 학생용 작업이 **구글시트 + Apps Script**로 돌아가는데, **API 쿼터(429)·Apps Script 채점 깨짐·시트 용량/깨짐** 오류가 잦다. 이를 **Django DB로 이전**하는 것이 이 작업의 큰 목표. (`writing` 앱이 이미 구글 의존 없이 XP·대전·라이브모니터까지 구현된 선례)

### 왜 Django
- 자체 DB → API 한도 없음 / 서버 파이썬 채점 → Apps Script 수식 대체 / 데이터 보호(학생은 화면만) / 통계·실시간 자유

### 새 데이터 흐름 (구글 제거)
```
완성(코드) 자동화 (엑셀·통합본 생성)   ← 선생님 입력(엑셀)은 유지 OK
      │  (gspread push 대신)
      ▼
Django import command / 내부 API
      ▼
Django DB  →  학생은 홈페이지에서 훈련·답지·채점
```
선생님 입력(엑셀)은 그대로, **학생용 출력만 구글시트 → Django**. 구글은 제거하거나 선생님 입력 보조로 축소.

### 이전 대상 (기능별로 단계적, 한 번에 X)
1. **단어훈련** (이 문서, 1단계) — 내신단어 → vocab 앱
2. **답지/채점** — 정답 Part1~4 + 서버 채점 → Apps Script 완전 대체
3. **오답·리포트·단어관리**

기존 구글 파이프라인(완성(코드) 쪽 `내신단어업로드.py`, `엑셀답지생성/업데이트.py`, 개별단어장생성.py, Apps Script)을 하나씩 Django 기능으로 대체하면 오류가 줄어든다. 단어훈련을 이 패턴(자동화→import→DB→학생화면)의 첫 사례로 만든다.
