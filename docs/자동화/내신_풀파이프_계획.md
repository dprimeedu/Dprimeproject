# 내신 풀파이프 — K. TEST 자동화 허브 신규 탭 계획

## 배경

현재 내신 부교재 출력 흐름은 5단계로 분리돼 있고, 각 단계마다 GUI 를 다시 열고 같은 파일을 재선택해야 함.

```
마스터 부교재
  ↓ (1) 학교 xlsx 생성        ← 부교재 출력 툴 (엑셀만 모드)
동백고2 통합.xlsx
  ↓ (2) 영어분석지            ← AI 자동화 툴
  ↓ (3) 문제 생성 + 내신단어 업데이트  ← AI 자동화 툴
  ↓ (4) (옵션) 변형문제 단어 백필  ← AI 자동화 툴 (예정)
  ↓ (5) 부교재 출력 (전체)     ← 부교재 출력 툴
한글/PDF 결과물
```

각 단계 결과는 모두 `동백고2 통합.xlsx` 한 파일에 누적됨. `copy_excel()` 이 이미 있는 파일은 패스하므로 데이터 보존됨.

## 목표

K. TEST 자동화 허브에 새 탭 **"내신 풀파이프"** 를 추가해, 학교명/시험명/시험범위만 입력하면 위 5단계를 한 화면에서 컨트롤.

## UI 스케치

```
[내신 풀파이프 탭]
┌─────────────────────────────────────────────────┐
│ 학교명: [동백고2          ]                       │
│ 시험:   [1학기 기말고사 ▼]                        │
│ 마스터 부교재 xlsx: [파일 선택]                    │
│   → 자동 계산된 작업 파일 경로:                    │
│      Z:\...\동백고2\동백고2 통합.xlsx              │
│                                                  │
│ 시험범위 (단원): [3-3,4-4,5-3,5-6,...        ]   │
│   ↑ 시험범위.txt 가 있으면 자동 로드                │
│                                                  │
│ 실행 단계:                                        │
│   ☑ 1. 학교 xlsx 생성 (부교재 출력 엑셀만 모드)    │
│   ☑ 2. 영어분석지                                 │
│   ☑ 3. 문제 생성 + 내신단어 업데이트              │
│   ☐ 4. 변형문제 단어 백필                         │
│   ☑ 5. 부교재 출력 (전체 옵션)                    │
│                                                  │
│ [순차 실행]                                       │
└─────────────────────────────────────────────────┘
[로그 출력 창]
```

## 핵심 설계

### 경로 매핑 규칙

`학교명` + `시험` 으로 작업 폴더 자동 계산:
```
Z:\home\Drive\교재폴더\내신\{학기 폴더}\{학교명}\{학교명} 통합.xlsx
```

예) `동백고2` + `1학기 기말고사` →
`Z:\home\Drive\교재폴더\내신\2026년 1학기 기말고사\동백고2\동백고2 통합.xlsx`

학기 폴더 — 현재 년도 + 학기로 자동 계산하거나, 첫 실행 시 폴더 선택 후 기억.

### 단계별 호출 방식

K. TEST 자동화 가 이미 subprocess 호출 패턴이라 그대로 활용:
- 각 단계 = 기존 .py 파일을 subprocess 로 실행
- 단계 사이는 직렬 (한 단계 끝나야 다음 시작)
- 각 단계가 입력 인자로 작업 xlsx 경로를 받게 수정 (현재는 GUI 에서 파일 선택 → 인자로 받도록 우회 필요)

### 인자 전달

각 도구가 CLI 인자로 작업 xlsx 경로 + 옵션을 받을 수 있어야 함. 현재 도구들은 GUI 기반이라 추가 작업 필요:
- `B. 부교재 출력.py` — CLI 모드 추가 (or 자동 시작 인자)
- `f-2. Gemini,Claude자동화.py` — CLI 모드 추가
- 백필 스크립트 — 처음부터 CLI 친화적으로

또는 더 가벼운 방안: 각 도구 GUI 를 띄우되 작업 파일/옵션을 자동 채워주고 "작업 시작" 만 자동 클릭. 비교적 침습 적음.

### 단계 부분 실행

체크박스로 단계 선택 가능 → 사용자가 단계별 결과 검토 후 다음 단계 진행하는 패턴 지원.

예) 처음엔 1번만 → AI 자동화 결과 검토 → 2-3번 → 검토 → 5번

## 곁다리 개선 (선택)

1. **시험범위 파일화**
   - `{학교 폴더}/시험범위.txt` 에 단원 적어두면 모든 도구가 읽어 쓰기
   - 매번 단원 입력 안 해도 됨
   
2. **마지막 사용 파일 기억**
   - AI 자동화 / 부교재 출력 둘 다 직전 작업 파일을 기억해 자동 선택
   - 풀파이프 탭이 부담스러우면 이것만으로도 마찰 절반 줄어듦

3. **진행률 표시**
   - 로그창에 단계별 시작/완료 시각 + 처리 행 수 표시
   - 실패 시 어느 단계에서 멈췄는지 명확하게

## 구현 우선순위 (이 md 적용 시)

1. 변형문제 단어 백필 — AI 자동화 새 탭으로 먼저 (현재 작업)
2. 백필 기능 안정화 후 → 풀파이프 탭 설계 본격화
3. 시험범위 파일화 / 마지막 사용 파일 기억 (선택)
