# 개발 환경 셋업 (Python conda edu)

> 이 문서는 **컴퓨터 단위** 환경 셋업 가이드입니다.
> 컴퓨터를 새로 셋업하거나 다른 사람이 이 프로젝트를 이어받을 때 따라하세요.
> 코드 자체와는 별개로, 어떤 Python 환경에서 코드를 실행할지를 정의합니다.

## 0. 결론 한 줄 요약

- **conda env `edu`** 가 주력 Python 환경. 모든 자동화 코드를 여기서 돌립니다.
- 시스템 Python (`AppData\Local\Programs\Python\...`) 은 더 이상 사용하지 않습니다.
- PowerShell 새 세션을 열면 `edu` 가 **자동 활성화** 되도록 셋업 완료.
- VSCode 워크스페이스 (`완성(코드)\`) 도 자동으로 edu 인터프리터를 사용.

## 3. PowerShell 자동 활성화 셋업

### 3-1. 한 번만 수행할 셋업 단계

```powershell
# (1) conda init 실행해서 PowerShell hook 등록
conda init powershell

# (2) ExecutionPolicy 풀기 (CurrentUser 스코프만, RemoteSigned 가 안전한 기본값)
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force
```

### 3-2. profile.ps1 끝에 추가

파일 경로: `C:\Users\<계정>\Documents\WindowsPowerShell\profile.ps1`
(없으면 위 `conda init powershell` 이 만들어 줌)

추가할 라인:
```powershell
# 새 PowerShell 세션에서 자동으로 edu 환경 활성화
conda activate edu
```

`#region conda initialize` 블록 **뒤에** 추가합니다. (hook 이 먼저 등록되어야 activate 가 동작)

### 3-3. 동작 확인

새 PowerShell 창 열고:
```powershell
python --version    # → edu 환경의 python 가리켜야 함
$env:CONDA_DEFAULT_ENV    # → "edu" 출력
```

## 4. VSCode 워크스페이스 자동 활성화

`완성(코드)\.vscode\settings.json` 에 이미 설정되어 있음:

```json
{
    "python.defaultInterpreterPath": "C:\\Users\\PrimeeduClassMain\\miniforge3\\envs\\edu\\python.exe",
    "python.terminal.activateEnvironment": false,
    "python.terminal.activateEnvInCurrentTerminal": false,
    "terminal.integrated.env.windows": {
        "PYTHONIOENCODING": "utf-8"
    }
}
```

**다른 컴퓨터로 옮길 때**: `python.defaultInterpreterPath` 값을 그 컴퓨터의 edu python 경로로 수정. 다른 항목은 그대로.

이렇게 하면:
- 이 워크스페이스 열면 자동으로 edu 인터프리터로 디버깅/실행됨
- 새 통합 터미널 열면 profile.ps1 이 `conda activate edu` 실행함 (VSCode 자동 활성화는 끔 — 켜두면 bare `activate` 가 주입돼서 base 로 떨어짐)
- 한글 stdout 깨짐 방지 (`PYTHONIOENCODING=utf-8`)

## 5. 시스템 Python 정리 기록 (2026-05-15 수행)

이 컴퓨터(`PrimeeduClassMain`) 에서는 시스템 Python (`AppData\Local\Programs\Python\Python310`) 에 third-party 패키지가 200개+ 쌓여 있었습니다. edu 와 거의 중복이라 일괄 삭제:

```powershell
$sysPy = "C:\Users\PrimeeduClassMain\AppData\Local\Programs\Python\Python310\python.exe"
# 1) 백업 (복원 시 사용 가능)
& $sysPy -m pip freeze > "$env:USERPROFILE\Desktop\sys_python_packages_backup_$(Get-Date -Format yyyyMMdd_HHmmss).txt"
# 2) 일괄 삭제
& $sysPy -m pip uninstall -y -r "백업파일경로"
```

결과: 시스템 Python 에는 `pip` + `setuptools` 만 남음.

**복원 필요 시** (어떤 외부 도구가 시스템 Python을 쓰고 있어서 깨졌다면):
```powershell
& "C:\Users\PrimeeduClassMain\AppData\Local\Programs\Python\Python310\python.exe" -m pip install -r "<백업파일>"
```

백업 파일은 바탕화면에 `sys_python_packages_backup_YYYYMMDD_HHMMSS.txt` 형태로 있음. 일정 기간 후 문제 없으면 삭제해도 됨.

## 7. 자주 묻는 트러블슈팅

### `conda activate` 시 "Run 'conda init' before 'conda activate'" 오류
→ §3-1 의 `conda init powershell` 부터 실행하세요.

### 새 PowerShell 열어도 edu 가 활성화 안 됨
→ ExecutionPolicy 가 Restricted 일 가능성. `Get-ExecutionPolicy -List` 확인.
→ `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned -Force` 로 풀기.

### `python` 명령이 여전히 시스템 Python 을 가리킴
→ PATH 환경변수에 시스템 Python 이 conda 보다 앞에 있을 수 있음.
→ profile.ps1 의 `conda activate edu` 가 실행되면 PATH 가 갱신되니, 새 PowerShell 세션을 열어보세요. 그래도 안 되면 PATH 점검.

### `conda run -n edu python …` 으로 한글 출력 시 cp949 에러
→ `conda run` 은 stdout encoding 제어 문제 있음. **직접 호출** + `$env:PYTHONIOENCODING="utf-8"` 사용:
```powershell
$env:PYTHONIOENCODING="utf-8"
& "C:\Users\PrimeeduClassMain\miniforge3\envs\edu\python.exe" your_script.py
```
PowerShell 프로필이 edu 를 활성화시켜 두면 그냥 `python your_script.py` 도 OK.

### VSCode 인터프리터 변경이 반영 안 됨
→ VSCode를 한 번 완전히 닫고 다시 열어야 워크스페이스 settings.json 의 인터프리터 설정이 적용됩니다.
→ Ctrl+Shift+P → "Python: Select Interpreter" 에서 수동으로도 변경 가능.

### VSCode 터미널 열자마자 `(edu)` → `(base)` 로 떨어짐
→ VSCode 의 `python.terminal.activateEnvironment` 가 켜져 있으면 터미널 시작 직후 bare `activate` 스크립트를 자동 주입합니다. bare `activate` = `conda activate base` 라서 profile.ps1 이 켜준 edu 가 base 로 덮어써집니다.
→ §4 settings.json 처럼 `activateEnvironment` 와 `activateEnvInCurrentTerminal` 을 모두 `false` 로 두세요. 환경 활성화는 profile.ps1 이 단독으로 담당합니다.

### 터미널 프롬프트가 `PS>` 가 아니라 그냥 `>` 로 시작 (cmd.exe 임)
→ profile.ps1 은 PowerShell 전용이라 cmd.exe 에서는 자동 활성화가 안 됩니다.
→ VSCode 기본 셸을 PowerShell 로 고정: settings.json 에 `"terminal.integrated.defaultProfile.windows": "PowerShell"` 추가.

### `프라임에듀.exe` 런처에서 자식 .py 실행 시 `ModuleNotFoundError: No module named 'win32com.client'` (또는 다른 패키지)
→ 원인: `프라임에듀.exe` 는 2023년에 PyInstaller 로 번들된 .exe 라 GUIABC.py 변경이 안 먹힘. 자식 .py 를 `subprocess.Popen(['python.exe', ...])` 로 띄우는데, Windows 의 `where python.exe` 가 시스템 Python(§5 정리로 빈 상태)을 1순위로 잡아서 import 실패.
→ 진짜 원인 위치: **Machine(시스템) PATH 의 1순위에 시스템 Python 디렉터리가 박혀 있음**. User PATH 수정만으로는 못 이김 (Effective PATH = Machine PATH + User PATH).
→ 해결: **관리자 권한 PowerShell** 에서 Machine PATH 의 시스템 Python 항목 제거 + edu prepend.

```powershell
# 관리자 PowerShell. 백업부터 뜨고 진행.
$mp = [Environment]::GetEnvironmentVariable("Path", "Machine")
$mp | Out-File "$env:USERPROFILE\Desktop\machine_path_backup_$(Get-Date -Format yyyyMMdd_HHmmss).txt" -Encoding utf8

$filtered = ($mp -split ';' | Where-Object {
    $_ -ne "" -and $_ -notlike "*Programs\Python\Python310*"
}) -join ';'
$eduDirs = "C:\Users\PrimeeduClassMain\miniforge3\envs\edu;C:\Users\PrimeeduClassMain\miniforge3\envs\edu\Scripts"
[Environment]::SetEnvironmentVariable("Path", "$eduDirs;$filtered", "Machine")

# Explorer 재시작 후 .exe 재실행
Stop-Process -Name explorer -Force
```

확인 방법: 새 cmd 창에서 `where python.exe` 첫 줄이 `...\envs\edu\python.exe` 로 나와야 함.

### `cp949 codec can't encode character '—'` (또는 다른 비-cp949 문자) 오류
→ 자식 Python 의 stdout 인코딩이 cp949 인 채로 print 가 em dash(—), 화살표(→), 일부 한자 등을 못 찍어서 터짐. 프라임에듀.exe 같은 런처는 PYTHONIOENCODING 을 자식에게 안 넘겨주는 경우가 있음.
→ 해결: **사용자 환경변수에 `PYTHONIOENCODING=utf-8` 영구 등록** → 모든 새 Python 프로세스가 자동으로 utf-8 stdout.

```powershell
[Environment]::SetEnvironmentVariable("PYTHONIOENCODING", "utf-8", "User")
# Explorer 재시작 후 .exe 재실행
Stop-Process -Name explorer -Force
```
