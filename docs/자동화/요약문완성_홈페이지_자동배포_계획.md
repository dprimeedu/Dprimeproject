# 요약문완성 → 홈페이지 자동 생성·배포 (학생별) — 계획 [HOLD, 2026-06-08]

> 상태: **보류**. 사용자가 "내신 오류 수정 먼저" 하느라 잠깐 홀드. 재개 시 이 문서대로 진행.
> 기존 기반 파이프라인은 [요약문완성훈련_synology_pipeline.md](요약문완성훈련_synology_pipeline.md) 참고.

## 0. 목표
지금까지 **구글 스프레드시트**에 학생별로 출제하던 "요약문완성"(빈칸 ⓐ·ⓑ, 내신단어 품사변형 정답)을
이제 **홈페이지(dprimeedu.synology.me 요약문완성훈련 앱)** 에서 학생별로 자동 생성·배포한다.

옛 출제물 형태(구글시트 예: `장시은 2026년 백현고2 1학기 중간고사 요약문완성`):
- 컬럼: 번호 / 문제(빈칸 `*____` 2개) / 답1·답2 / 단원(1-1,1-3…) / 정답1·정답2 / 한글1·한글2 / O·X 체크
- **정답1/2** = 인정 정답(기본형), **답1/2** = 요약문에 실제 나타난 굴절형, O/X = 둘 일치 여부 플래그
- 번호 1~10 = 지문(원문 색인) 범위

## 1. 트리거 / 입력 데이터
- **단어관리자료.xlsx** (`TEST 및 답지\일괄출력\TEST 자동화\단어관리자료.xlsx`, active 시트)
  - 행 컬럼(0-based): C(idx2)=교재명, D(idx3)=이름, E(idx4)=시작, G(idx6)=끝
  - **교재명 == "요약문완성"** 인 행만 대상. 이름=학생, 시작~끝 = **원문 색인 범위(지문 번호)**
  - 학생마다 범위 다를 수 있음 → **학생별 개별 단원**(사용자 결정)
- **학교 조회**: `0. 학생관리자료.xlsx` 학생Data 시트 (A=이름, C=학교학년). 이름으로 학교학년 찾음.
- **시즌(고정, 사용자 확인)**: **2026년 1학기 기말고사**.

## 2. 통합본 경로 (글롭 자동 탐색)
원문 + 내신단어 시트가 든 학교 통합본:
```
Z:\home\Drive\교재폴더\내신\{학년폴더}\{개정본|개정판}\2026년\2026년 1학기 기말고사\{학교}\{학교} 1학기 기말고사 통합.xlsx
```
- `{학년폴더}` 예: 고2→`고등학교 2학년`, 고1→`고등학교 1학년`. `개정본`(고2)/`개정판`(고1) **학교마다 다름** → 글롭으로 찾을 것.
- part2 작업에서 검증된 글롭 방식(여러 깊이 와일드카드, `Get-ChildItem -Path` 패턴)을 재사용. 전체 `-Recurse`는 NAS에서 20s 타임아웃 → 와일드카드 경로가 빠름.

## 3. 현재 코드 상태 (재개 전 파악 완료)
### 생성 측 — 완성(코드)
- `CLASS/Gemini.py:1153` `create_summary(target_file, target_mode, push_web=True, school=None)`
  - `원문` 시트 row 2..끝 **전체 순회**(색인/원문/단원 컬럼). 지문마다 `generate_summary` → `요약문완성` 시트 기록 + `web_items` 누적.
  - 끝에 `push_summary(school or _school_from_filename, web_items)`.
  - **현재 한계: 지문 범위(시작~끝) 없음, 학생 개념 없음, unit=단원(1-1 등) per item.**
  - `Claude.py`는 Gemini 상속(프롬프트/generate_summary 공유).
- `CLASS/요약문웹동기화.py` : `summary_to_item(created, idx, unit)` (현재 item['unit']=단원), `push_summary(school, items)` → `{token, school, items}` POST.
- GUI: `_AI자동화_원본/f-2. Gemini,Claude자동화.py` "요약문 생성" 탭(수동 파일 선택 + push_web 체크박스).

### 웹 측 — Dprimeproject (summary 앱)
- `summary/views.py:494` `import_api` : body `{token, school, unit?, items:[{idx,unit?,sub_unit?,sentence1_template,sentence1_answer,korean1,sentence2_template,sentence2_answer,korean2}]}`.
  - (school, unit)별 그룹 → `SummaryUnit` upsert → `SummaryProblem` 전체 교체(replace-on-reimport). **배정 없음.**
  - URL `POST /training/summary/api/import/`, 토큰 `pedu-summary-2026`.
- 모델: `SummaryUnit`(school,unit unique) / `SummaryProblem`(unit,index, sentence1/2_*, korean1/2, sub_unit) / `SummaryAssignment`(student,unit unique; **student=User pk로만 배정, login_id/username 조회 없음**).

## 4. 구현 계획
### A. 웹 (Dprimeproject) — 학생 배정
- `import_api`에 옵션 `assign_to`(학생 이름 또는 login_id) 추가 → 단원 생성/갱신 후 그 학생에게 `SummaryAssignment.get_or_create`.
  - 학생 매칭: writing/vocab의 `_find_student`(username 우선, login_id 보조) 패턴 재사용.
- **학생별 단원**: 푸시 시 `unit` = 학생 식별자(예: `문준혁 1학기 기말고사`), 문제별 `sub_unit` = 단원(1-1 등).
- main 푸시 → NAS 자동배포(~1분). **모델 변경 없음 → migrate 불필요.**

### B. 생성 (완성코드)
- `create_summary`에 **지문 범위(색인 start~end)** + **student/assign** 파라미터 추가(범위 밖 색인 skip).
- `summary_to_item`/`push_summary`: `unit`=학생식별자, `sub_unit`=단원, payload에 `assign_to`=학생 추가.
- **새 배치 드라이버**(예: `CLASS/요약문배치.py` 또는 GUI 핸들러):
  1. 단어관리자료에서 `교재명==요약문완성` 행 수집 → [(이름, 시작, 끝)]
  2. 이름→학교(학생관리자료) 조회
  3. 통합본 글롭 탐색(2번 경로)
  4. `create_summary(통합본, start, end, school, student, assign_to=student, push_web=True)`
- E. AI 자동화 "요약문 생성" 탭에 **"단어관리자료 자동"** 버튼/옵션 추가(또는 별도 실행 스크립트).

### C. 테스트 우선 (비용·품질)
- **문준혁(백현고2) 1명만** 먼저 생성 → 홈페이지에 학생 단원 생성 + 그 학생 배정 + 빈칸 정답 품질 육안 확인.
- 이상 없으면 단어관리자료 전체 행 일괄.

## 5. 열린 항목 / 주의
- `generate_summary`가 지문당 AI 1회 호출 → **비용**. 범위/학생 수 곱만큼 발생. 1명 테스트로 단가 확인 후 전체.
- 동명이인 시 학생 매칭 모호 → login_id 병행 고려(단어관리자료엔 이름만 있음).
- 재푸시 = (school, unit=학생) 단위 덮어쓰기라 중복 안 쌓임(import replace-on-reimport).
- 토큰 변경 시 NAS `.env SUMMARY_IMPORT_TOKEN` + `요약문웹동기화.py TOKEN` 양쪽.
