## 1. 진단 — 장시은 어법 170문제 "또 만들라" 현상
- **결론:** 데이터는 멀쩡히 저장됨. `수능특강 light 영어(2025) 통합.xlsx` 변형문제 시트에 어법 `[변형2]` 정확히 170개.
- **원인:** 취약유형보충의 생성/카운트가 **둘 다 시험범위(단원) 필터**를 검. 어제 시험범위 미해석으로 `units=None`(전 단원)으로 생성돼 170개가 65개 단원에 분산 → 오늘 시험범위가 좁게 잡히면 그 단원 밖은 안 세져 "부족"으로 뜸.
- 판별: ① 분석 로그의 `■ 이 학생이 풀어야 할 단원(시험범위)` 줄 확인. ⚠ 전체풀 메시지면 units=None.

## 2. 진단·수정 — 장시은이 part4로 나온 문제
- **원인:** `merge_pdf`(제본)의 권 번호 계산이 학교 폴더 `next_part_number`(이름 붙은 폴더 무시)와 **다른 방식** — 제본 폴더의 같은 학교 PDF 최대 Part +1인데 **학생 이름 붙은 개별 PDF도 카운트**해서 문준혁 Part3 → 장시은 Part4가 됨.
- **수정:** `고등부교재자동화.py merge_pdf` — `name_suffix`(개별 학생) 있으면 제본 폴더 스캔 대신 `self.NAME`에 박힌 part 번호 사용.
- 잘못 생긴 장시은 출력물(제본 Part4 + 학교폴더 part3 + 답지) 전부 삭제 후 재작업 대기.

## 3. 참고 — 요약문 생성 소스 시트
- `create_summary`(요약문생성)는 통합본 **`원문` 시트** 지문을 읽어 생성. 정답 단어는 **`내신단어` 시트** 품사변형 풀.

## 4. 함정 메모 — VS Code Run 버튼 KeyboardInterrupt
- AI 생성 도는 중 다른 .py Run 누르면 같은 터미널 재사용 → Ctrl+C로 기존 프로세스 죽임(=KeyboardInterrupt). 코드 버그 아님. 동시 작업은 새 터미널에서.

## 6. 신규(대규모) — 부교재/교과서 출력 ④ 홈페이지(웹앱) 업로드

**웹 import API 신설·배포(migration 없음):**

- 내신단어: `vocab/words_import_api` = `/training/vocab/api/words/import/`. 커밋 9e85c23.
- (요약문은 기존 `push_summary` 재사용)

**데스크톱:** CLASS/영작웹동기화.py·내신단어웹동기화.py + 요약문은 sheet_row_to_item 역변환기. `create_summary`에 `unit_filter` 추가.

**부교재 출력 배선:** `_runWritingUpload`/`_runVocabWebUpload`/`_runSummaryWebUpload` (후처리) — 마스터 시트를 **시험범위(selectedUnit)로 필터 → push + 학교 학생 전원 자동배정**(`_webAssignStudents`). 요약문은 'AI생성' 옵션(기본 ON) 켜지면 없는 단원 create_summary 생성→누적→push, 끄면 있는 것만.

**교과서 출력에도 동일 적용:**

- 중등 교과서(l-1 중3 / l-2 중1,2): **영작만** (구조가 달라 중요영작 HWP→영작 웹앱 전환만).

**검증:** 영작·내신단어 실푸시 end-to-end OK, 학생 자동배정 OK(동백고2 8명). 교과서 통합본 시트는 아직 데이터 비어있어 실푸시는 GUI 실행 시 확인.
