## 1. 확정된 결정 사항

| 항목 | 결정 |
|---|---|
| **A. 콤보 리셋 기준** | 단어 콤보 + 문장 콤보 둘 다 트래킹 |
| **B. 레벨 호칭** | 새내기 → 견습생 → 영작러 → 영작러+ → 영작마스터 → 영작신 |
| **C. 소리 효과** | 1차 구현 X (구조만 잡아둠, 나중에 추가 가능) |
| **D. 리더보드** | 본인 순위 + 1~3등만 공개 (메달 카드로 화려하게) |
| **E. 레벨업 보상** | 호칭 + 배지 모음 (테마 변경 등은 X) |
| **F. 시도 횟수** | 3회 (1차 + 한글뜻 힌트 + 첫글자 힌트 + 정답 공개) |
| **G. 엑셀 업로드** | admin에 파일 업로드 기능 |

## 2. 점수 시스템

### 단어별 기본 점수
| 정답 시점 | 점수 |
|---|---|
| 1차 시도 정답 (힌트 없이) | **+10점** |
| 1번 힌트(한글뜻) 후 정답 | +6점 |
| 2번 힌트(첫글자) 후 정답 | +3점 |
| 3번 힌트(정답 공개) — 학습 인정 | +1점 |

### 보너스 점수
| 조건 | 보너스 |
|---|---|
| Speed Bonus (단어 3초 안 정답) | +2점 |
| Perfect Sentence (한 문장 전체 무실수) | +20점 |
| 단어 콤보 5/10/20/50 도달 | +10/+20/+40/+100점 |
| 문장 콤보 3/5/10 도달 (연속 perfect sentence) | +50/+100/+200점 |
| 첫 일일 로그인 | +50 XP |

### XP / 레벨 공식
- **XP = 누적 점수** (1:1)
- **레벨 = floor(XP / 100) + 1**

### 호칭 매핑
| 레벨 | 호칭 |
|---|---|
| Lv 1-2 | 새내기 |
| Lv 3-5 | 견습생 |
| Lv 6-10 | 영작러 |
| Lv 11-20 | 영작러+ |
| Lv 21-50 | 영작마스터 |
| Lv 51+ | 영작신 |

## 3. DB 모델

### 새 앱: `writing/`

```python
# 1. 단원
class WritingUnit:
    title          # 단원명 (예: "고1 진단평가 3과")
    publisher      # 출판사 (옵션)
    grade          # 학년 (옵션)
    description    # 설명 (옵션)
    created_at
    created_by     # 선생님 (FK to Member)

# 2. 문제
class WritingProblem:
    unit           # FK to WritingUnit
    index          # 단원 내 번호
    korean         # 한글 문장 + 힌트
    english        # 영어 정답 (전체 문장)
    word_hints     # [선택] 각 단어의 한글뜻 (콤마 구분)
                   # 없으면 시스템이 단어별 한글뜻 못 줌

# 3. 한 회차 풀이 세션
class WritingSession:
    student        # FK to Member
    unit           # FK to WritingUnit
    started_at
    finished_at    # null이면 진행 중
    total_score    # 이 세션에서 획득한 점수
    perfect_sentences  # 무실수 문장 개수
    max_combo      # 이 세션 최고 콤보

# 4. 단어 단위 시도 로그
class WritingAttempt:
    session        # FK to WritingSession
    problem        # FK to WritingProblem
    word_index
    input_value
    correct_answer
    hint_level     # 0(노힌트)/1(한글뜻)/2(첫글자)/3(정답공개)
    is_correct
    attempt_num
    time_taken_seconds
    score_earned
    created_at

# 5. 게임화 프로필 (학생당 1개)
class StudentProfile:
    student        # OneToOne to Member
    total_xp       # 누적 XP
    level          # 현재 레벨 (XP로 계산)
    title          # 현재 호칭 (레벨로 계산)
    current_word_combo
    max_word_combo_ever
    current_sentence_combo
    max_sentence_combo_ever
    last_login_date
    login_streak_days

# 6. 배지 정의
class Achievement:
    code           # "combo_50"
    name           # "콤보 마스터"
    description
    icon           # 이모지 또는 파일 경로
    condition_type # "combo" / "perfect_count" / "streak" / "speed" 등
    condition_value

# 7. 학생-배지 매핑
class StudentAchievement:
    student
    achievement
    earned_at
```

## 4. URL 구조

```
/training/                       → 영작훈련 홈 (단원 리스트)
/training/unit/<id>/             → 단원 선택 → 세션 시작
/training/session/<id>/          → 풀이 중 화면 (핵심 UI)

/training/leaderboard/           → 리더보드
/training/badges/                → 배지 모음

/training/admin/units/           → 단원 관리 (선생님)
/training/api/check-word/        → POST: 단어 채점 (AJAX)
/training/api/use-hint/          → POST: 힌트 사용 (AJAX)
```

## 5. 채점 API 설계

### POST /training/api/check-word/
**Request**:
```json
{
  "session_id": 123,
  "problem_id": 45,
  "word_index": 2,
  "input": "under",
  "hint_level": 0
}
```

**Response**:
```json
{
  "is_correct": true,
  "score_earned": 10,
  "current_word_combo": 5,
  "current_sentence_combo": 1,
  "total_score": 145,
  "total_xp": 145,
  "level": 2,
  "level_up": false,
  "badges_earned": [],
  "next_hint_available": null  // 정답이라 힌트 더 필요 없음
}
```

### POST /training/api/use-hint/
**Request**:
```json
{
  "session_id": 123,
  "problem_id": 45,
  "word_index": 2,
  "current_hint_level": 0
}
```

**Response**:
```json
{
  "new_hint_level": 1,
  "hint_content": "아래에",  // 한글뜻 (또는 첫글자 "u" 등)
  "hint_type": "korean_meaning",
  "remaining_score": 6
}
```

## 9. 보안/UX 디테일

### 정답 유출 방지 (협업자가 짚은 핵심 이슈)
- 클라이언트엔 절대 정답을 보내지 않음
- AJAX로 단어 채점 → 서버에서 비교 → boolean + 점수만 응답
- 힌트도 서버에서 단계별로 발급

### 입력 정규화
```python
def normalize(text):
    return text.lower().strip().replace(",.?!", "")
```

### 시도 횟수 검증
- 클라이언트에서 카운트 + 서버에서도 카운트 (조작 방지)
- 3회 초과 시 서버가 거부

### 동시 사용
- 학원 PC 공용 사용 가능
- 학생별 로그인이므로 서로 영향 없음

## 10. 엑셀 파일 형식 (선생님 가이드)

| 컬럼명 | 필수 | 예시 |
|---|---|---|
| 색인 | ✅ | 1, 2, 3... |
| 한글 및 힌트 | ✅ | "3년도 채 안 되어 프로젝트는 끝났다 (under / finish)" |
| 영어 | ✅ | "In under three years was the project finished" |
| 단어별 한글뜻 | 🟡 옵션 | "안에,~아래,셋,년,...,프로젝트,끝났다" (콤마 구분) |

→ "단어별 한글뜻" 없으면 시스템이 첫 번째 힌트("한글뜻 표시") 못 줌. 영어 단어 자체 첫글자 힌트(2단계)와 정답 공개(3단계)는 작동.
→ 진짜 잘 만들려면 선생님이 단어별 한글뜻도 입력. 시간 절약 위해 처음엔 비워둬도 OK.

## 11. 확정된 추가 결정 사항 (2026-05-16 추가)

- **학생 계정**: 자유 가입 (학생이 회원가입 후 자기가 등록)
- **단원 할당**: 선생님이 학생별로 배정
- **단어별 한글뜻**: AI 자동 생성 (OpenAI 또는 Anthropic Claude API 사용)
  - 엑셀 업로드 시 백엔드에서 AI 호출 → 단어별 한글뜻 자동 생성
  - 결과는 DB에 저장 (생성은 업로드 시 1회만)
  - admin에서 선생님이 검토/수정 가능
  - 예상 비용: 단원 1개 (10~30문장) 업로드당 약 1~5원

## 12. AI API 설정 (옵션 A 채택)

### 작동 방식
```
[엑셀 업로드] → [백엔드에서 AI API 호출] 
              → [영어 단어별 한글뜻 매핑 받음]
              → [DB의 WritingProblem.word_hints 컬럼에 저장]
              → [퀴즈 풀 때 사용]
```

### 필요한 것
- API 키 (OpenAI 또는 Anthropic 중 선택)
- 키는 `.env` 파일에 저장 (git 제외)
- 모델: 가벼운 거 (gpt-4o-mini / claude-haiku) — 정확도 충분, 비용 1/10

### 채택 모델: **Google Gemini 2.0 Flash**
- 무료 한도: 분당 15 요청, 일 1500 요청 (학원 규모에 충분)
- 유료 시: 1M 토큰당 약 $0.075 (가장 저렴)
- 한국어 처리 우수
- Python 패키지: `google-generativeai`
- API 키 발급: https://aistudio.google.com/app/apikey

### 환경변수 (`.env`)
```
GEMINI_API_KEY=your_key_here
```
