# 영작훈련 시스템 개발 계획

> dprimeedu.synology.me 에 영작훈련 기능을 추가하는 작업 계획서
> 작성일: 2026-05-16

---

## 1. 개발 범위 (확정)

클라이의 작동 원리를 차용해서 dprimeedu에 자체 구현. 콘텐츠는 본인 자료(영작 시트, 직접 입력) 사용.

### ✅ 구현할 기능

1. **시스템 전체 설계** — 클라이의 좋은 흐름을 dprimeedu에 구현
2. **DB 모델** — 단원/문제/정답/학습로그 구조
3. **시험 유형 4가지**:
   - 영작 (한글 → 영어 전체 문장)
   - 단어 빈칸 채우기 (Apps Script 패턴)
   - 어순 배열
   - 뜻 쓰기 (영어 → 한글)
4. **채점 로직** — 정규화(구두점/대소문자 제거) 후 비교
5. **학습 로그 분석 화면** — 학생별 약점, 시간, 시도 횟수
6. **인쇄·PDF 출력** 기능
7. **선생님용 admin** — 단원·문제·정답 입력 화면

### ❌ 안 함

- 클라이 콘텐츠 자동 수집 스크래퍼
- 클라이 코드 직접 복붙
- 클라이 로고/상표/디자인 그대로 사용

## 2. 정보 출처

| 출처 | 본인 접근 | AI 접근 |
|---|---|---|
| 클라이 admin/사용자 화면 | ✅ 본인 계정으로 다 봄 | ❌ (스크린샷 받아야 함) |

## 3. 발견 프로세스 5단계

### 1단계: 클라이 화면 둘러보기

클라이 admin과 학생 화면을 모든 메뉴 클릭해보고 스크린샷.

**찍어야 할 화면**:

- 메인 진입 화면
- 단원 선택 화면 (이미 봄)
- 시험 유형 선택 (영작/뜻쓰기/어순배열)
- 시험지 미리보기 (이미 봄)
- **학생이 실제로 푸는 화면** ← 중요, 아직 못 봄
- **채점 결과 화면** ← 중요, 아직 못 봄
- **선생님이 학생 학습 통계 보는 화면** ← 중요
- **단원·문제 입력하는 화면**
- **인쇄물 실제 출력 PDF 모양**

스크린샷 → AI에게 보내면 UI 패턴 추출 후 디자인에 반영.

### 2단계: 흐름(flow) 문서화

클라이/Apps Script 둘 다 흐름을 문장으로 정리.

**[학생 흐름] 답해야 할 질문**:

1. 로그인 (학생 ID 입력? 자동?)
2. 본인이 풀 단원이 자동 표시? 선생님이 미리 배정?
3. 시험 시작 → 문제 한 번에 다 보임? 한 문제씩?
4. 단어별 채점? 문장 전체 채점?
5. 다 풀고 나면 점수만? 약한 단어 강조?
6. 결과가 선생님한테 어떻게 보이는지?

**[선생님 흐름] 답해야 할 질문**:

1. admin 로그인
2. 학생이 본 단원 선택
3. 학생별 통계 보기 → 어떤 정보(점수/시간/약점 단어)
4. 새 단원/문제 추가하는 흐름

### 3단계: "우리 식 결정"

클라이/Apps Script와 다르게 갈 부분 정하기.

**결정 사항**:

- **학생 식별**: 이름 텍스트 입력 vs Django 로그인?
  - 클라이는 별도 ID, Apps Script는 이름 입력
  - 우리는?

- **채점 단위**: 단어별(Apps Script) vs 문장 전체(클라이 영작)?
  - 둘 다 시험 유형으로 제공?

- **재시험**: 학생이 같은 단원 여러 번 풀 수 있나?
  - 시도 회차 기록만 늘리기 vs 새 시도로 분리

- **공용 PC**: 학원에서 같은 PC 쓰는 학생들 구분 어떻게?
  - 매번 이름 입력 vs 학생 로그인

- **결과 공개**: 채점 결과를 학생이 즉시 보나? 선생님만 봐?

### 4단계: 데이터 단위 추출

"이런 정보가 어디 저장되어야 하나?" 추출.

**[단원]**
- 단원명 (예: "천재 중2 03과 - Wrap Up")
- 출판사 (천재/YBM/...)
- 학년 (중2/고1/...)
- 단원 번호 (3과)
- 부분 (Wrap Up / Reading / ...)

**[문제]**
- 어느 단원에 속하나
- 문제 번호 (1~8)
- 한글 문장
- 영어 정답
- 힌트
- 난이도? (선택)

**[학습 로그]**
- 누가 (학생)
- 언제 (timestamp)
- 어느 문제
- 시도 회차
- 입력값
- 정답 여부
- 소요 시간

→ 이게 곧 Django 모델 정의. 3단계 결정 끝나면 거의 자동 도출.

### 5단계: AI에게 던지기 → Django 구현

본인이 1~4단계 정리해서 주면 AI는:

- Django 모델 만들기
- 마이그레이션 작성
- 뷰/URL 구조 설계
- 템플릿 구현
- 채점 API 구현
- admin 화면 커스터마이징
- 인쇄/PDF 생성

## 4. 본인이 다음에 할 일 (우선순위 순)

### A. 스크린샷 10장 (5~10분)

클라이의 다음 화면들을 캡처:

1. 학생이 시험 푸는 실제 화면 (3가지 유형 각각)
2. 채점 결과 화면
3. 선생님이 학생 학습 통계 보는 화면 (있다면)
4. 단원 추가 화면 (있다면)

### B. 결정 사항 답하기 (10분)

위 3단계의 질문들에 한 줄씩 답:

- 학생 식별 방식?
- 채점 단위?
- 결과 공개 방식?
- 같은 PC 학생들 구분?
- 재시험 정책?

### C. 영작 시트 다운로드 (5분)

구글 시트를 `.xlsx`로 받아서 프로젝트 폴더에 두기.

---

## 5. 진행 방식

작은 단위로 진행. **A만** 먼저 시작해도 OK. 스크린샷 5장 정도부터 시작해서 점차 채워가는 게 현실적.

각 단계마다 AI가 적극적으로 질문 → 본인이 답하면서 진도 나가는 방식.

## 6. 참고 자료

### 이미 확보된 자료

- Apps Script 코드 (doget.gs + index.html) — 본인이 만든 원래 영작 도구
- 영작 시트 구조 (3컬럼: 색인, 영어, 한글 및 힌트)
- 클라이 admin 일부 스크린샷 (단원 리스트, 시험 유형 선택, 미리보기)

### 관련 파일

- 기존 Apps Script: 단어별 빈칸 채우기 + 즉시 채점 + 3회 시도 + 학습로그 자동 기록
- 영작 시트: 약 27행 (실제는 더 있을 수 있음), 6.4MB

### 채점 정규화 규칙 (Apps Script 기준)

```javascript
cleanInput = studentVal.replace(/[.,?!]/g, "").toLowerCase()
```

- 구두점 제거 (`.,?!`)
- 소문자 통일
- 완전 일치 비교

### 학습 로그 컬럼 (Apps Script 기준 9개)

1. 일시
2. 학생이름
3. 문항
4. 단어순번
5. 입력값
6. 정답
7. 결과 (O/X)
8. 시도회차
9. 문항소요시간

## 7. 법적 안전 가이드라인

- ✅ 작동 원리, UI 흐름, 기능 차용 — 합법
- ✅ 본인이 만든 콘텐츠 사용 — 합법
- ✅ 본인이 교과서 보고 직접 타이핑 → admin 수동 입력 — 회색지대지만 학원가 관행
- ❌ 클라이 콘텐츠 자동 추출 — 계약 위반
- ❌ 클라이 코드/디자인 복붙 — 저작권 위반

콘텐츠 입력은 본인 책임 영역, AI는 시스템(콘텐츠 중립적 도구)만 만듦.
