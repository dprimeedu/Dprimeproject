## 1. 원본 데이터

- **위치**: `Z:\home\Drive\TEST 및 답지\일괄출력\TEST 자동화\어법\` (G 시리즈, 79개 파일)
- **로드 코드**: `완성(코드)\권용국 작업\단나누기+OCR\개별단어장생성.py` `process_grammar_data()` (B/C/D열 읽음)
- **배정 관리**: `단어관리자료.xlsx` 한 행 = (책이름에 "어법" 포함 → 어법모드, 학생, 시작~끝 범위)

### 형식 2종 (열 구성이 다름 — 통합 필요)
| 변형 | 예시 파일 | 열 구성 | 정답 형태 |
|---|---|---|---|
| **내신어법** | G구성고1내신어법2단계, G동백고1… | A=번호 · D=문장 · E=정답 · F=단원(3-1) | **`O`**(맞음) 또는 **`wrong->right`**(고칠 것, 예 `had->has`) |
| **모의고사어법** | G고등어법12/34/45 | A=번호 · B=문장(`[..]`/`①~⑤`) · C=구분 · D=정답 | 교정형(`To avoid, Avoiding`) 또는 번호(`1`) |

- 문장 안의 어법 포인트는 **`[대괄호]`** 또는 **`①~⑤`** 로 표시.
- 내신어법 = "이 문장 어법상 맞나? 틀리면 고쳐라" (O / 교정).
- 학교 내신이 핵심이므로 **내신어법(O / wrong->right)** 우선.

## 2. 응시/채점 방식 — ⚠️ 확정 필요

샘플 본 뒤 후보(택1 또는 혼합):

- **(A) O / 고쳐쓰기 (주관식)** — 학생이 각 문장에 "맞음(O)" 또는 "고친 형태" 입력.
  - 채점: `O`면 학생 O 여부, 교정문제면 학생입력 vs 정답의 `->오른쪽` 비교(정규화). **자동채점 가능**.
- **(B) 틀린 부분 번호 고르기 (객관식)** — `①~⑤` 표시 문장에서 틀린 번호 선택. 모의고사어법 변형에 맞음. 자동채점.
- **(C) 교사 채점 (요약문 방식)** — 학생 입력 → 교사가 O/X. 가장 단순/안전하지만 수기.

> 권장: **(A) 자동채점** 기본 + 애매한 건 교사 보정(요약문 채점 UI 재사용). 모의고사 ①~⑤ 문제는 (B)로.

## 3. 웹 앱 구조 (vocab/summary 패턴 그대로)

신규 앱 **`grammar`** (`/training/grammar/`).

### 모델 (`grammar/models.py`)
- `GrammarUnit` — school·exam·title·grade·is_active (vocab/summary와 동일 패턴, 학교토큰 포함)
- `GrammarProblem` — unit·index·sentence(문장)·answer(정답 원문 `O`/`wrong->right`)·corrected(파싱된 정답)·marker_type(bracket/circled)·sub_unit(3-1)
- `GrammarSession` — student·unit·start_index·end_index·status(진행/제출/채점)·correct_count·total
- `GrammarAnswer` — session·problem·student_input·is_correct/admin_verdict (요약문 BlankAnswer 패턴)
- `GrammarRangeTest` — student·unit·start_index·end_index·source_label·pass_threshold (단어 RangeTest 패턴 — 오늘 볼 어법TEST)
- `GrammarAssignment` — student·unit (writing/summary와 동일)

### import API (`/training/grammar/api/import/`, 토큰)
- 단어(`words_import_api`)·요약문(`import_api`) 패턴. items=[{idx, sentence, answer, sub_unit}], school, mode(replace/merge/append).
- 로컬 푸시 도구: `어법웹동기화.py` 신규(요약문웹동기화.py 복제) — `process_grammar_data` 결과를 전송.
- 범위(오늘 볼 어법TEST): `range_import_api` 패턴 — 학생관리표 '어법TEST' 열에서 동기화.

### 학생 응시 (`grammar/views.py` + templates)
- 홈: 배정 단원 + 10문제(or N)씩 청크 버튼 (요약문 home 패턴)
- 응시: 문장별 [맞음 O] / [고쳐쓰기 입력칸] (또는 ①~⑤ 선택). 자동채점 → 결과.
- 채점 보정: 교사용 채점 목록/상세(요약문 grade_list/grade_detail 재사용 구조).

### 자동배정 · 현황판
- 업로드 시 `member.auto_assign` 재사용(학교토큰 매칭 → 그 학교 학생 자동배정).
- `report/study.py` board에 **어법 칸** 추가(현재 단어/요약문/영작/시험 4칸 → 5칸) + 오늘 볼 어법TEST 범위/합격.

## 4. 채점 로직 (자동, 내신어법 기준)
- 정답 `O` → 학생이 "맞음" 표시하면 정답.
- 정답 `wrong->right` → 학생입력을 정규화 후 `right`(화살표 오른쪽)와 비교. 일치=정답.
  - 다중 허용(`To avoid, Avoiding`)은 콤마 분리 후 하나라도 일치하면 정답.
- 애매/주관 = 교사 보정(요약문 O/X 토글 재사용).
