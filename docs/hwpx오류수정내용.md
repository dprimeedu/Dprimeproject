# HWPX 변형문제 출력 — 오류 수정 내용 정리 (전체)

> 변형문제(모의고사 변형) HWPX/PDF 다운로드의 렌더·표기 오류와 수정 기록 **전체**.
> 기간: **2026-05-30 ~ 2026-06-25**. 최신 갱신: 2026-06-25.

## 파이프라인 (어디를 고치면 되나)

```
엑셀/DB(ModifiedQuestions_Data)
   → common/text_postprocess.py   (텍스트 정규화: 마커·밑줄·구분자·발문)
   → common/hwpx/hwpx_builder.py  (HWPX 조판/렌더: charPr·박스·단 넘침)
      ↘ PDF 경로(WeasyPrint, HTML→PDF) — academy/views.py 의 별도 렌더
   → .hwpx / .pdf  → (HWPX는) 한컴 COM 으로 PDF 변환해 눈으로 검증
```

- 오류는 **두 층**에서 난다.
  1. **렌더·조판 층**(`hwpx_builder.py`, PDF 경로): charPr 정렬·박스·줄간격·단 넘침 등.
  2. **텍스트 정규화 층**(`text_postprocess.py`): 마커·밑줄·구분자·발문 표기.
- **정본 규칙 = `common/text_postprocess.py`**, 로컬 'AI 자동화' 미러 = `완성(코드)/CLASS/text_postprocess.py`.
  **둘은 항상 같이 고쳐 동기화**(웹 다운로드와 로컬 엑셀 후처리가 같은 양식 보장).
- 정규화는 **렌더 시점**에 적용 → 이미 올라간 데이터도 **재업로드 없이** 다음 출력부터 교정.

## 1. 렌더·조판 층 (hwpx_builder.py / PDF)

### 1-1. ★charPr 위치=id 정렬 (핵심)  — `118f8d50` (06-25)
- **증상**: 발문이 굵게가 아니라 **밑줄**로 나오고, 지문 밑줄이 깨짐.
- **원인**: HWPX 의 `charPrIDRef` 는 charPr 의 **id 값이 아니라 `charProperties` 배열의 '위치(인덱스)'** 를 가리킨다. 배열이 밀리면 "굵게" 자리를 다른 글자모양이 차지해 엉뚱하게 렌더.
- **수정**: charPr 정의를 **배열 위치 = 의도한 id** 가 되도록 정렬해 주입.
- **검증법**: 한컴 COM 으로 `hwpx → pdf` 변환 후 **눈으로** 확인(발문 굵게 / 지문 밑줄 / 날짜 파랑).
- 직전 시도(`ff4a57c1`)와 그 되돌림(`b53ccc19`)을 거쳐 이 커밋에서 정리됨.
- 참고: `charPr11`=날짜(굵게+파랑), `charPr12`=발문(굵게)/지문 밑줄(BOTTOM), `charPr8`=일반.

## 4. 재발 방지 체크리스트
1. **charPrIDRef 는 '위치'다** — charPr 정의 순서가 밀리면 굵게↔밑줄이 뒤바뀐다. 빌더 수정 시 항상 한컴 COM PDF 로 눈 검증.
2. **두 미러 동기화** — `common/text_postprocess.py` 를 고치면 `완성(코드)/CLASS/text_postprocess.py` 도 같이 고친다.
3. **밑줄 유지 유형** — 지칭대상/지칭추론·밑줄의미·함축의미·어법·어휘만 지문 밑줄 유지. 그 외는 잔여 밑줄로 보고 제거.
4. **두-빈칸 판정** — `___(A)___`(밑줄로 둘러싼 빈칸)만 두-빈칸. 순서의 문단 라벨 `(A)(B)(C)` 와 혼동 금지.
5. **단 넘침은 추정** — 한/글 실제 조판과 다를 수 있어 보수적으로. 정밀 조판은 한/글에서 한 번 열어 확인.
6. **렌더 단계 처리** — 정규화는 출력 시 적용되므로 데이터 재업로드 없이 즉시 반영. 운영은 push 후 cron 자동배포(~1분).
