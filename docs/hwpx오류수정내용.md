# HWPX 변형문제 출력 — 오류 수정 내용 정리

> 변형문제(모의고사 변형) HWPX/PDF 다운로드에서 나온 렌더·표기 오류와 그 수정 기록.
> 최신 갱신: **2026-06-25**

## 파이프라인 (어디를 고치면 되나)

```
엑셀/DB(ModifiedQuestions_Data)
   → common/text_postprocess.py   (텍스트 정규화: 마커·밑줄·구분자·발문)
   → common/hwpx/hwpx_builder.py  (조판/렌더: charPr·박스·단 넘침)
   → .hwpx  → 한컴 COM 으로 PDF 변환해 눈으로 검증
```

- 오류는 **두 층**에서 난다.
  1. **렌더 층**(`hwpx_builder.py`): charPr 정렬·박스·줄간격 등 한/글 XML 조판.
  2. **텍스트 정규화 층**(`text_postprocess.py`): 마커·밑줄·구분자·발문 표기.
- **정본 규칙 = `common/text_postprocess.py`**, 로컬 'AI 자동화' 미러 = `완성(코드)/CLASS/text_postprocess.py`.
  **둘은 항상 같이 고쳐 동기화**한다(웹 다운로드와 로컬 엑셀 후처리가 같은 양식을 보장).
- 정규화는 **렌더 시점**에 적용된다 → 이미 올라간 데이터도 **재업로드 없이** 다음 출력부터 교정된다.

## 1. 렌더 층 (hwpx_builder.py)

### 1-1. ★charPr 위치=id 정렬 (핵심)  — `118f8d502` (2026-06-25)
- **증상**: 발문이 굵게가 아니라 **밑줄**로 나오고, 지문 밑줄이 깨짐.
- **원인**: HWPX 의 `charPrIDRef` 는 charPr 의 **id 값이 아니라 `charProperties` 배열의 '위치(인덱스)'** 를 가리킨다. 배열 순서가 밀리면 "굵게(11/12)" 자리를 "밑줄(12)" 등이 차지해 엉뚱하게 렌더된다.
- **수정**: charPr 정의를 **배열 위치 = 의도한 id** 가 되도록 정렬해 주입.
- **검증법**: 한컴 COM 으로 `hwpx → pdf` 변환 후 **눈으로** 확인(발문 굵게 / 지문 밑줄 / 날짜 파랑).
- 참고: `charPr11`=날짜(굵게+파랑), `charPr12`=발문(굵게)/지문 밑줄(BOTTOM), `charPr8`=일반.

### 1-2. 조판 미세조정 (참고)
한 단 채움·페이지 경계·줄간격 등은 오류라기보다 조판 튜닝 이력이다(되돌린 것 포함).
- 줄간격 160% 통일, `lineseg`(조판 캐시) 제거로 줄 겹침 방지 — `common/hwpx/README.md` 참고.
- 발문↔지문 박스가 단/페이지로 분리되지 않게 보호.
- 짧은 선택지도 한 줄에 합치지 않고 각 줄에 분리.
- 단 넘침은 **추정**이라 한/글 실제 조판과 100% 일치하지 않을 수 있음(보수적으로 설정).

## 3. 재발 방지 체크리스트
1. **charPrIDRef 는 '위치'다** — charPr 정의 순서가 밀리면 굵게↔밑줄이 뒤바뀐다. 빌더 수정 시 항상 한컴 COM PDF 로 눈 검증.
2. **두 미러 동기화** — `common/text_postprocess.py` 를 고치면 `완성(코드)/CLASS/text_postprocess.py` 도 같이 고친다.
3. **밑줄 유지 유형** — 지칭대상/지칭추론·밑줄의미·함축의미·어법·어휘만 지문 밑줄을 유지. 그 외는 잔여 밑줄로 보고 제거.
4. **두-빈칸 판정** — `___(A)___`(밑줄로 둘러싼 빈칸)만 두-빈칸. 순서의 문단 라벨 `(A)(B)(C)` 와 혼동 금지.
5. **렌더 단계 처리** — 정규화는 출력 시 적용되므로 데이터 재업로드 없이 즉시 반영. 운영은 push 후 cron 자동배포(~1분).
