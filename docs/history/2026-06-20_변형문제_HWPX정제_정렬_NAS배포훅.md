## A. 변형문제 보기 텍스트 정제 (academy/views.py)

DB 원본 → HWPX/PDF 변환 과정에 데이터별 형식이 너무 들쭉날쭉해서 한꺼번에 잡았다.
세 경로(HTML 미리보기 `_split_choices`, PDF `_split_choices`, HWPX `_hwpx_choices`)
모두 동일 규칙 적용.

### A-1. 짧은 선택지 한 줄에 합쳐버리는 버그
`build_question_block` 의 "모두 30자 미만이면 한 단락으로 join" 최적화 때문에
`① supervision  influenced② interference  protected…` 처럼 마커들이 줄 없이 붙어
나왔다. 그 분기 통째로 제거 → 각 선택지는 항상 별도 단락.

### A-2. 엑셀 인라인 그림 placeholder
openpyxl 이 셀에 박힌 그림을 `icon_1_3` 같은 텍스트로 가져왔다.
`_normalize_breaks` / `_hwpx_clean` 양쪽에 `re.sub(r'icon_\d+_\d+', '', s)` 추가.

### A-3. 마커 자동 분리가 출제자 의도 깨던 케이스
순서 유형 원본이 `① (A)-(C)-(B)② (B)-(A)-(C)③ (B)-(C)-(A)\r\n④ … ⑤ …` 처럼
줄바꿈이 부분적으로만 들어있는데, 코드가 `①②③` 마커 앞에서 다 쪼개버려 5줄로 펼침.
→ 엑셀 원본 줄바꿈만 신뢰하도록 자동 분리 로직 제거.

### A-4. 같은 줄 마커 사이 가독성
A-3 의 부작용으로 `① (A)-(C)-(B)② (B)-(A)-(C)…` 같이 마커가 붙는 케이스 발생.
`_space_inline_markers` — `②~⑩` 앞 글자가 공백이 아니면 공백 3개 자동 삽입.
줄 시작 `①` 은 건드리지 않음.

### A-5. 두-빈칸 유형 단어 사이 `……` 일괄 표시
요약문완성 / 연결어 / 연결사 데이터가 어떤 행은 `단어A \t…… \t단어B`,
어떤 행은 `단어A \t단어B`, 어떤 행은 `단어A …… 단어B` 식.
`_normalize_two_blank` — `qtype` 이 두-빈칸 유형이면 단어 사이 구분자를
` …… ` 로 통일 (tab 단독·2칸+공백·기존 `……`/`...`/`⋯` 모두 인식).

함정 하나 — 처음 만들 때 `_hwpx_clean` 이 tab 을 공백 1개로 바꾼 다음에
`\s{2,}` 만 찾아서 `① coach\tmislead` 케이스는 매치 못 했다.
`_hwpx_choices` 가 `_hwpx_clean` 우회하고 정규식이 `\t` 단독도 분리자로 인식하도록 수정.

### A-6. 잘못 섞인 숫자 prefix 방어
`[일치불일치]` 한 행(pk_number 7862 idx 19) 옵션 셀에 `174 ③ Practically …`
처럼 회차/페이지 번호가 텍스트로 들어가 있었음 → 엑셀에서 수정 + DB 도 직접
`'174 ③' → '③'` replace + `_strip_marker_garbage` 로 코드 측 방어
(`^\d+\s+(?=[①-⑩])` 패턴, 다른 동일 오타에도 동작).

## B. 변형문제 출력 정렬 — 부교재 자동화 표준 도입

이전: `order_by('pk_number', 'index')` — 같은 회차의 모든 변형이 연속으로 몰림.
B. 부교재 출력 (`CLASS\고등부교재자동화.py::excel_question`) 의 정렬 의도를 그대로 옮김.

### 유형 그룹 (`__typeCheck` 와 동일)
| 그룹 | 멤버 |
|---|---|
| TYPE1 | 순서·문장넣기·무관한문장·연결어·연결사 |
| TYPE2 | 일치불일치·기타 |
| TYPE3 | 지칭대상·지칭추론 |
| TYPE4 | 주제·주장·제목·요지·목적·밑줄의미·함축의미 |
| TYPE5 | 어휘 |
| TYPE6 | 요약문완성·빈칸 |
| 그 외 | 도표·그림·심경분위기·어법·서술형·장문·장문2·장문3 |

### 라운드로빈
같은 유형 안에서 `total_number` 별로 1개씩 pop → 다음 회전에 또 1개씩.
즉 `1회차-1번째변형 → 2회차-1번째변형 → 3회차-… → 1회차-2번째변형 → …`
같은 번호가 몰리는 현상 해소.

### 화면 chip 순서도 표준에 맞춤
`PREFERRED` 리스트를 `Question.TYPE_LIST_ALL` (부교재 자동화) 순서로 교체.
누락됐던 `[기타]` (일치불일치 다음) / `[서술형]` (어법 다음) 추가.

세 경로 모두 동일 `_round_robin_sort(rows, keytable_map)` 사용 — HTML 미리보기·PDF·HWPX.

## C. NAS 자동배포 훅 + gunicorn 운영

### Stop 훅이 git pull 만 하고 워커 리로드를 안 함
"코드 푸시했는데 그대로인데??" 의 원인. 워커가 옛 코드 들고 있던 거.
gunicorn 이 `--pid /tmp/gunicorn.pid` 로 띄워져 있어서 HUP 시그널로 무중단 리로드 가능.

`.claude/settings.json` Stop 훅 명령에 추가:
```
… && [ -f /tmp/gunicorn.pid ] && kill -HUP $(cat /tmp/gunicorn.pid)
```

### ⚠ `docker restart django` 금지
gunicorn 은 컨테이너 PID 1 이 아니라 `docker exec -d` 로 띄운 비-PID1 프로세스.
컨테이너 재시작하면 같이 안 살아남고 → 502. 이번에 한 번 겪었고, 메모리에 못박음.
코드 리로드는 **반드시** HUP, 컨테이너 재시작이 정말 필요하면 재시작 후 즉시 가동 명령 재실행.

### 운영 메모리 갱신
`reference_nas_infrastructure.md` 에 위 두 항목 + 가동 명령 명시.

## D. 협업자 푸시로 깨진 logout 복구

토요일 협의 전 협업자가 소셜 로그인(`django-allauth`) 작업을 main 에 푸시.
`config/urls.py` 의 `path('accounts/', include('django.contrib.auth.urls'))` →
`include('allauth.urls'))` 로 교체.

allauth 는 `account_logout` 이름만 제공 → `base.html` 두 곳의 `{% url 'logout' %}` 가
`NoReverseMatch` 일으켜 **모든 페이지 500**.

`config/urls.py` 에 LogoutView 를 `name='logout'` 로 직접 재등록해 복원.
(템플릿 일괄 수정 대신 URL alias — 협업자 변경 영향 최소화)

## E. 메모 — 토요일 협업자와 정리할 것

- gunicorn 가동 방식 일치: 항상 `--workers 3 --pid /tmp/gunicorn.pid` 옵션 포함
  (그래야 Stop 훅 HUP 이 동작)
- `allauth.urls` vs Django auth URL 이름 정책 — 템플릿 일괄 `account_*` 로 갈 것인가 둘 다 노출할 것인가
- 변형문제 데이터 정합성 — 엑셀 셀에 페이지번호·아이콘 placeholder 가 섞이는 입력 워크플로 개선
