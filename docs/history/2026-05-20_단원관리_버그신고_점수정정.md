## 1. 단원 상세 페이지 전면 강화

`/training/writing/admin/units/<id>/` — 단원 1개 안의 문제 목록을 보고 편집하는 페이지.

### 인라인 편집 (한글/영어 셀)
- 한글/영어 셀 클릭 → 그 자리에 `textarea` 등장 → 편집
- `Ctrl+Enter` 저장 / `Esc` 취소 / blur 시 자동 저장
- 저장 성공 시 셀이 잠시 녹색 플래시
- 엔드포인트: `problem_update` POST `{korean?, english?, word_hints?}` 중 보낸 필드만 갱신

### 단어 뜻 모달 (Quizlet 풍 카드)
- 표의 `✅ N` 배지 클릭 → 카드형 모달 오픈
- 각 카드: `[번호] [단어 입력] [뜻 입력] [고유명사 토글] [≡ 드래그] [🗑 삭제]`
- 카드 사이 hover 시 보라 원형 `+` 버튼 → 그 위치에 빈 카드 삽입
- 드래그 핸들 mousedown 시에만 카드 draggable=true (입력 클릭과 충돌 X)
- "+ 단어 추가" / "🪄 영어에서 자동 채우기" (영어 정답에서 단어 추출, 중복 제외)
- 저장 시 표의 ✅ N 배지 카운트 자동 갱신
- 엔드포인트: `problem_hints_get` GET / `problem_update` POST {word_hints}

### 표 + 버튼 (행 사이 새 문제 삽입)
- 표 안 행과 행 사이 좁은 공간 hover → 보라 점선 가이드 + 가운데 `+`
- 클릭 → 그 위치에 **빈** 문제 행 생성 → reload + 그 행으로 스크롤 + 노란 강조 1.5초
- 한글/영어는 **완전히 빈 셀**로 시작 (placeholder 문구 제거 — 사용자 요청)
- 엔드포인트: `problem_insert` POST `{after_index}` — 인덱스 자동 재정렬

### 행 드래그 + 휴지통 (관리 컬럼)
- 우측 새 "관리" 컬럼: `≡ 드래그` / `🗑 휴지통`
- 드래그 핸들 mousedown → 행 draggable=true → 다른 행 위 드래그 → 상단/하단에 보라 선 가이드
- drop → 즉시 서버에 순서 저장 + 번호 자동 재부여
- 휴지통 클릭 → confirm → 단일 문제 + 풀이 기록 삭제 + 번호 재정렬
- 엔드포인트: `problem_reorder` POST `{order: [pid1, pid2, ...]}`

### 체크박스 + 일괄 삭제
- 표 왼쪽 새 체크박스 컬럼 (헤더 전체선택 포함)
- 선택 시 표 위 보라 툴바 등장: 전체선택/전체해제/`🗑 선택 삭제`
- 엔드포인트: `problems_delete` POST `{problem_ids}`

### 엑셀 업로드 (단원 통째 교체)
- "한글뜻 생성 완료" 옆 `📊 엑셀 업로드` 버튼
- 모달: 파일 선택 + 양식 다운로드 링크 + 경고문 + JS confirm
- 동작: 기존 problems 전부 삭제 → 파싱한 행으로 재생성 (3단어 이하 자동 제외)
- AI 한글뜻 초기화 / 배정·풀이 기록은 유지
- 엔드포인트: `unit_replace_excel` POST multipart

### AI 한글뜻 — 보충 (N개) / 재생성
- 라벨 개선: `🪄 AI 한글뜻 보충 (N개)` (N = `word_hints=[]` 인 문제 수)
- 모두 완료된 상태: `✅ 한글뜻 완료 (재생성)` outline 버튼 → confirm 후 force=True
- `generate_hints_ajax` 가 `body.force=true` 받아 모든 word_hints 초기화 후 재생성

## 2. 학생 ↔ 단원 배정 UX 재설계

### 학생 관리 페이지 — 학생 클릭 시 모달
- `/training/writing/admin/students/` 에서 학생 행 클릭 → 단원 배정 모달
- 백엔드: `student_assignments` GET / `student_assignments_update` POST

### 모달 패턴 (1차 — 단일 ✕ 버튼)
- 처음엔 토글(`배정된 단원만 / 전체 단원`) + 행마다 ✕ 단일 해제 버튼

### 모달 패턴 (2차 — 선택 체크박스 + 일괄)
사용자 요청: "✕ 단일 해제가 아니고 선택 삭제 패턴으로". 문제 표와 동일하게:
- 행 좌측 별도 **선택 체크박스** (배정 상태와 분리)
- 행 우측 **배정 상태 배지** (✓ 배정중 / 미배정)
- 1개 이상 선택 시 보라 툴바: `전체선택 / 전체해제 / ➕ 선택 추가 / 🗑 선택 삭제`
- 즉시 API 호출 → DB 반영 → 학생 행의 배정 카운트 자동 갱신
- 헤더 우측 표시: `배정중 N / 표시 N` (선택 카운트는 보라 툴바에 별도)

### 단원 상세 페이지 → "배정 학생: N명" 클릭
- 2단 모달:
  1. **배정 학생 목록** — 이 단원에 배정된 학생만 (학생 이름 + login_id + `›` 화살표)
  2. **학생별 단원 편집** — 1단 닫고 위와 동일한 패턴의 단원 편집 모달
- 현재 보고 있는 단원은 노란 배경 + `현재 단원` 배지로 강조
- `← 학생 목록으로` 버튼으로 1단 복귀
- 저장 후 이 단원에서 빠진 학생이 있으면 페이지 헤더의 "배정 학생: N명" 자동 갱신

## 5. 풀이 화면 UX 개선

### GREAT / GOOD / BOO 라벨 아래 차감 점수 표시
사용자 요청: "학생이 점수 깎인 이유를 즉시 납득하게". PERFECT(10점) 기준:
- PERFECT: 표시 없음 (만점)
- GREAT: **-4점** (실제 +6)
- GOOD: **-7점** (실제 +3)
- BOO~: **-9점** (실제 +1)
- 작고 굵은 빨간 글씨로 라벨 바로 아래

### 자동 채우기 — 콤마 숫자 지원
- 기존 `_NUMBER_PATTERN`은 `\d+` 만 매칭 (7000 OK, 7,000 X)
- `_NUMBER_WITH_COMMA_PATTERN = ^\d{1,3}(,\d{3})+(st|nd|rd|th|s)?$` 추가
- 매칭: `7,000` / `1,500,000` / `7,000s` / `12,000th` ✓
- 비매칭: `7,00` / `7,0000` ✗ (말 안 되는 콤마 위치)

## 6. 점수 시스템 버그 수정

### 32.5점 사건 — 고유명사가 만점 분모에 포함
사용자 신고: "PERFECT + GOOD인데 왜 32.5점?"

분석:
- 문장: `Kim Deuksin was a painter` (5단어)
- 클라이언트 자동채우기: Kim, Deuksin (`proper_noun=true`), a (관사) → 3개
- 학생 입력: was(PERFECT +10), painter(GOOD +3) = 13점
- **버그**: `complete_problem_api` 의 `sentence_max` 계산이 `_should_auto_fill` 만 사용 → Kim/Deuksin을 non-auto로 카운트 → 만점 = `4 × 10 = 40` → 13/40 = **32.5%** (불합격)

수정:
- `_is_auto(i, w)` 헬퍼 추가 — `_should_auto_fill(w, i) OR word_hints[i].proper_noun`
- 동일 문장 → 만점 = `2 × 10 = 20` → 13/20 = **65%** (정확)

`check_word_api` 의 자동채우기 검증 로직(`is_proper or _should_auto_fill`)과 일치하게 됨.

## 8. 발견·고친 인프라 이슈

### Bootstrap 모달 lazy-init 버그
증상: 단어뜻 모달이 열리지 않고 "배정 학생" 클릭도 무반응. 신규 기능 모두 작동 안 함.

원인: Bootstrap JS는 `base.html` 하단(line 122)에서 로드되는데 `{% block content %}` 안 인라인 스크립트 최상단에 `new bootstrap.Modal(...)` 3개가 있어 `ReferenceError: bootstrap is not defined` → 그 뒤 모든 `addEventListener` 미등록.

해결: Modal 인스턴스를 lazy getter 함수로 감쌈 — 첫 호출 시에만 생성 (그때엔 Bootstrap이 이미 로드됨).
```js
let _hintsModal = null;
function hintsModal() {
    if (!_hintsModal) _hintsModal = new bootstrap.Modal(document.getElementById('hintsModal'));
    return _hintsModal;
}
```
모든 사용처 `hintsModal.show()` → `hintsModal().show()` 로 변경.

### 모달 aria-hidden + 포커스 접근성 경고
증상: 콘솔에 `Blocked aria-hidden on an element because its descendant retained focus` 빨간 경고.

원인: Bootstrap 5 버그 — 모달 닫기 버튼(.btn-close) 클릭 → 그 버튼이 포커스를 갖고 있는 채로 모달에 aria-hidden=true 부착 → 브라우저 경고.

해결: `base.html` 글로벌 핸들러 추가 — `hide.bs.modal` 이벤트 직전 모달 안 활성 요소 blur:
```js
document.addEventListener('hide.bs.modal', e => {
    if (e.target.contains(document.activeElement)) document.activeElement.blur();
}, true);
```

### 학생 모달 "전체 선택/해제" 무반응
원인: 옛 `.assign-unit-cb`(이제 hidden) 토글하고 있어 화면 변화 X. 새 `.select-cb`로 변경.

## 다음에 손볼 것

### 우선순위 높음
- **사용자 테스트** — 학생이 버그 신고 잘 쌓이는지 확인. 첫 며칠 운영하면서 신고 내용 보고 UX 개선
- **빈 셀 안전 가드** — `+` 로 만든 빈 한글/영어 셀이 풀이 화면에 노출되면 이상함. start_session에서 빈 문제 skip 또는 unit_detail에서 빈 셀 시각적 강조

### 우선순위 중간
- **자동채우기 확장 후보** — `$7,000` (통화 기호) / `9-11` (범위) / 분수 / 소수
- **단어뜻 모달 일괄 작업** — 현재 단원이 아닌 인접 문제들도 같이 편집할 수 있는 모드
- **부교재 출력 후속** — `.env` 운영 키 등록 / Dockerfile 형상 정리 (5/19 docs 참고)

### 우선순위 낮음
- 학생 본인 통계 페이지 / 단원별 합격률 / 배지 컬렉션 화면
- 단원 amount 정상 + 일부만 풀고 일시 정지 → 재개
