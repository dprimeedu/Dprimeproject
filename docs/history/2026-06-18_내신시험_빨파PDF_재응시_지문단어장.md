## 3. 시험 결과 화면 — 2차 오답만 기본 노출 + 점수 뱃지 토글

2차 채점완료된 시험은 1차 오답 22문제 중 16문제를 2차에서 맞춘 상태. 그런데도 표/빨파 카드에
1차 오답 22개가 다 보여서 진짜 오답(6개)을 찾기 어려움.

### 변경
- 2차 채점완료 시 `r2-ok` 행은 기본 `d-none` — 화면에는 "2차 오답 6개" 카드 + 표 6행만
- 별도 토글 버튼 대신 **점수 뱃지 자체가 토글**: `1차 33/469` 클릭 → 1차 오답 전체 표시,
  `2차 추가정답 16/22` 클릭 → 2차 오답만(기본)
- 활성 뱃지는 그림자로 강조, 비활성은 `opacity:.55` 로 흐리게
- 1차만 채점된 세션은 기존 동작 그대로(모든 1차 오답 노출, 토글 없음)

별도 액션 두 건도 같이 정리:
- 1차 오답 표에 **통합본 번호 컬럼(`ref_number`)** 추가 — 내신/유형별만 노출
  (예: `7-3`, `2026-03-24`). 모의고사는 ref가 비어 컬럼 자체가 안 보임
- 교사 화면 좌상단 back-link을 "시험 목록" → **"학습 현황판"** (학생은 기존 `exam:home`)

**관련 파일:** `exam/views.py`, `exam/templates/exam/result.html`

## 4. 내신 빨파 정답 PDF — 번호 클릭 → 새 탭 열기

오답 표의 번호 클릭 시 NAS '교재폴더'에 있는 해당 문항의 빨파 정답 PDF를 새 탭에서 바로 열게.

### 매핑 규칙
`ExamPaper.redblue_relpath(ref_number)` 가 다음 트리를 따라가 PDF를 찾는다:

```
/redblue_root/내신/{고등학교 N학년}/{개정판|개정본}/{YYYY년}/{YYYY년 1학기 기말고사}/
  {청덕고1}/{청덕고1 1학기 기말고사(part1)}/
  {... 정답}/{... 빨파정답}/
  {... N. {ref_number} 빨파 정답.pdf}
```

- `school_grade`(`청덕고1`/`동백고2`/`동백중2`) → 학년 폴더(`고등학교 1학년` / `중학교 N학년`)
- 판본은 학년별로 다름 — `개정판` / `개정본` 두 후보를 차례로 시도
- `season`("2026 1학기 기말") → 폴더명 정규화(끝에 "고사" 자동 부착)
- `category=Part1` → 폴더 suffix `(part1)`
- 파일 매칭은 폴더 안에서 ` {ref_number} `(양쪽 공백) 포함 PDF — `청덕고1 1학기 기말고사(part1) 9. 7-3 빨파 정답.pdf` 매칭 검증 완료

### 서빙
인증된 PDF 다운로드는 **nginx X-Accel-Redirect** 위임:

- `exam:redblue_pdf` view (`/training/exam/redblue/{question_id}.pdf`)
- 권한: 교사는 항상, 학생은 `session.redblue_released=True` 일 때만
- 응답 헤더 `X-Accel-Redirect: /redblue_internal/{percent-encoded-path}`
- nginx `location /redblue_internal/ { internal; alias /redblue_root/; }` 가 실제 스트리밍

**관련 파일:**
- `config/settings.py` — `REDBLUE_ROOT = '/redblue_root'`
- `exam/models.py` — `ExamPaper.redblue_relpath`
- `exam/views.py` — `redblue_pdf`, `result_view` (wrong[*].rb_pdf_url)
- `exam/urls.py` — `redblue/<int:question_id>.pdf`
- `exam/templates/exam/result.html` — 번호 셀 `target="_blank"`
- `nginx/default.conf` — `location /redblue_internal/`

**커밋:** `5258caf0`, `583d8305`

## 5. 답지 자동화 — 정답 엑셀 D열·E열 자동 출력

빨파 PDF 매칭과 응시 후 단어 클릭 동작은 둘 다 `ExamQuestion.ref_number` / `text` 가 채워져야
작동. 현재 답지 푸시는 `[번호, 정답, 유형]` 3-튜플만 보내고 있어 둘 다 빈 값.

답지 생성 단계(`고등부교재자동화.py`)에서 시험지 번호 ↔ 부교재 번호 ↔ 영어 지문이 한 함수
(`_insertQues`) 안에서 동시에 결정되는 점을 발견 — 거기서 두 리스트만 같이 누적하면 정답 엑셀의
D·E열로 자동 출력 가능.

### 자동화 코드 변경
- `_insertQues(source, ..., unitlist=None, englist=None)`:
  - `unit` ↔ `eng` 를 두 list 에 append (호출자가 list 전달 시에만)
- 변형문제 출력·내신TEST 출력 함수:
  - `unitlist = []` / `englist = []` 선언
  - `_insertQues` 호출에 두 리스트 전달
  - `createAnswerExcel(..., unitList=unitlist, engList=englist)` 로 넘김
- `createAnswerExcel`:
  - 헤더 `[색인, 정답, 유형, 원본번호, 지문]` (옵션 list 가 None 이면 해당 컬럼 생략)
  - D열 = `unitList[i]`, E열 = `engList[i]`

### 웹 측 푸시 흐름
- `B. 부교재 출력.py:_readAnswerItemsMulti`:
  - 정답 엑셀 D·E열을 읽어 5-튜플 `[번호, 정답, 유형, ref, text]` 로 반환
- `import_naesin_api._parse_question_item` 은 이미 list[3]=ref_number, list[4]=text 매핑 지원

→ 사용자분이 다음번 '변형문제'/'내신TEST' 출력 → '웹답지업뎃' 실행하면 모든 ExamQuestion 의
ref_number/text 가 자동으로 채워짐 → 빨파 PDF 링크와 지문 단어 클릭이 즉시 활성화.

**관련 파일:** (사용자 PC 자동화, git 외부)
- `Z:\home\Drive\문서 자동화 작업\완성(코드)\CLASS\고등부교재자동화.py`
  — `_insertQues`, 변형문제 출력, 내신TEST 출력, `createAnswerExcel`
- `Z:\home\Drive\문서 자동화 작업\완성(코드)\B. 부교재 출력.py`

## 9. 인프라·운영 변경

### nginx + Django 컨테이너 빨파 폴더 마운트
빨파 PDF 서빙을 위해 두 컨테이너 모두 NAS '교재폴더'를 read-only 로 마운트.

```
/volume1/homes/admin/home/Drive/교재폴더 → /redblue_root (ro)
```

- nginx: `docker stop nginx && docker rm nginx && docker run -d --name nginx ... -v 교재폴더:/redblue_root:ro --link django:django nginx:latest`
- Django: 동일 마운트 추가 + `-it` 옵션 유지

### Django 컨테이너 재생성 → 운영 다운 → 복구
Django 컨테이너를 재생성하니 컨테이너 내부에 누적되어 있던 `python3` 등 시스템 패키지가 사라져
gunicorn 기동 실패(HTTP 502). 기존 컨테이너가 ubuntu:24.04 base에 수동으로 apt-installed 한
상태였고, 그 변경이 이미지로 commit 되지 않았음.

복구:
- `docker exec django apt install -y python3 python3-pip` (venv 의 `python3` 심링크가 시스템
  `/usr/bin/python3` 를 가리킴)
- `docker exec -d django bash -c 'cd /home/Dprimeproject && exec venv/bin/gunicorn --bind 0.0.0.0:8000 --workers 3 --threads 2 --worker-class gthread --timeout 120 config.wsgi:application >> /tmp/gunicorn.log 2>&1'`
- HTTP 200 복귀, gunicorn 4 워커 정상

> **재발 방지**: 다음 컨테이너 재생성 전 `docker commit django dprime-django:base` 로 현재
> 컨테이너 상태를 이미지로 박제하거나, Dockerfile 에 `apt install` 단계 명시해서 재현 가능하게.

### 자동배포 훅 미동작
이번 푸시 4건(`583d8305`/`adda2cca`/`c0284a2c`/`0cfec819`)에서 자동배포 훅이 트리거되지 않아
매번 수동 동기화:

```bash
docker exec django bash -c 'cd /home/Dprimeproject && git pull && kill -HUP $(pgrep -f "gunicorn.*config.wsgi" | head -1)'
```

훅의 위치/조건은 미확인 — Synology Task Scheduler 또는 webhook 점검 필요.
