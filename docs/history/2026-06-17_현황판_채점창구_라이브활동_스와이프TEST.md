## 1. 시험 메뉴·결과·빨파 UX 정리

### 학생 시험 메뉴: 내 응시 결과 카드 추가
- `/training/exam/` 학생 분기에 본인 제출 세션 최근 30건을 카드 그리드로 노출
- 카드 한 장 = 시험지 1건. 1차 점수 / 2차 완료 / **빨파공개** 뱃지 + "결과 보기" 버튼
- 빨파 정답이 공개된 세션은 카드 테두리 자체를 초록색으로 — 한눈에 어떤 회차가 공개됐는지 보인다
- 학생용 "모의고사 빨파 — 회차 골라 응시" 빨간 버튼은 메뉴에서 제거 (URL 자체는 유지)

### 유형별(SOURCE_MOCK_TYPE) 2차 빨파 매핑
- 유형별 시험지는 여러 회차의 문항을 재번호(예: 193~224)해서 묶은 가상 시험지 — 기존 `redblue_qmap_for_mock` 는 `source==SOURCE_MOCK` 만 처리해 즉시 `{}` 반환, 2차 빨파 지문이 안 보였음
- 각 문항의 `ref_number='2014-09-31'`(원본 연도-강-번호)를 파싱 → 회차별 빨파 paper 조회 → 원본 번호로 ExamQuestion 매칭
- `result_view` 의 `is_mock` → `is_mock_like` 로 확장(MOCK + MOCK_TYPE 모두 빨파 시도). MOCK_TYPE 일 때는 단어 클릭 컨텍스트의 year/month=0(섞임)

### 교사 결과/채점 페이지 간소화
- 표는 **틀린 문항만** 표시 (`for a in answers` → `for a in wrong`). 맞은 행 숨김
- 표 위 안내: "번호 클릭 → 해당 문항 빨파 정답으로 이동"
- 번호 셀이 `<a href="#rb-{번호}">` — 페이지 아래 빨파 카드로 부드럽게 스크롤 + `:target` 노란색 깜빡임으로 강조
- 빨파 카드에 빨파 지문/[빨]/[파] 텍스트도 같이 표시 (선생님이 학생에게 즉시 설명 가능)

**관련 파일:**
- `exam/views.py` — `redblue_qmap_for_mock`, `_find_redblue_paper`, `result_view`, `student_home`
- `exam/templates/exam/home.html`, `exam/templates/exam/result.html`

## 2. 학습 현황판 — 채점 창구 통일

`/report/study/` 의 운영 목적을 "**누가 무엇을 제출했고 어디서 채점해야 하는지가 한 페이지에 보이는 허브**" 로 명확히.

- `config/context_processors.pending_badges` 에 `summary_pending` 추가(기존엔 grammar/exam 만)

### 시험 셀 클릭형 + 1차제출 라벨
- 기존엔 단순 라벨 + "채점" 뱃지 → **세션별 클릭 칩으로 통일** (요약문/어법 패턴과 동일)
- 색 구분:
  - 노랑: 학생 1차 제출 후 교사 미확인 (`XX점·1차제출` — 기존 "미확인" 문구 교체)

  - 파랑(연하게): 교사 확인 완료
- 각 칩의 href = `exam:result(session_id)` → 학생 결과/채점 페이지 직행

### 빈 세션 + 중복 시도 정리 (시험·어법)

## 3. 현황판 정렬 — 엑셀 학생관리표와 같은 요일+시간 순

기존: 학교학년 + 이름순. → 학생관리표 엑셀과 일치하지 않아 시간대별 수업 흐름이 안 보임.

### 정렬 알고리즘 이식
`Z:\home\Drive\TEST 및 답지\일괄출력\TEST 자동화\학생관리표_일일업데이트.py` 의 `parse_hour` 그대로 옮김:

```python
def _parse_class_hour(daytime_str, target_day):
    dstr = ''.join(c for c in daytime_str if c in _WD)  # '월수금'
    nums = ''.join(c for c in daytime_str if c.isdigit())  # '444'
    idx = dstr.find(target_day)                            # 수=1
    return int(nums[idx])                                  # nums[1]=4 → 4시
```

- 오늘 요일 = `'월화수목금토일'[date.weekday()]`
- 오늘 수업 학생을 위로(시간 오름차순), 그 외는 학교학년·이름 순

### 필요한 인프라 추가
- `StudentInfo.weekday_time` CharField 신규 (예: `'월수금444시'`). 마이그레이션 `report 0004`
- `/training/exam/api/import-student-schedule/` API 가 `daytime` 도 함께 받아 `weekday_time` 저장 (기존엔 weekdays 만)
- `학생요일동기화.py` `post()` 가 `daytime` 도 페이로드에 포함

### 활성화 (1회 실행)
```bash
python "Z:\home\Drive\TEST 및 답지\일괄출력\TEST 자동화\학생요일동기화.py"
```
- 33명 전원 동기화 완료 → 오늘(수요일) 수업 12명이 4시/6시/7시 순으로 위에 정렬됨 (엑셀과 동일)
- 새 학생 추가 / 수업 시간 변경 시 동일 스크립트 재실행

**관련 파일:**

## 4. 실시간 활동 표시 — 'XX 중 HH:MM:SS'

학생 이름 아래에 "지금 무슨 과목을 하고 있는지" + 경과 시간을 펄스 도트와 함께 표시.

### 방향 B: 기존 데이터 활용 (마이그레이션 없음)

| 과목 | 활동 판정 타임스탬프 |
|------|------------------|
| 영작 | `WritingSession.live_updated_at` — 타이핑 push (가장 정확) |
| 단어 | `VocabAttempt.created_at` Max via session |
| 요약문 | `SummaryBlankAnswer.updated_at` Max via session |
| 어법 | `GrammarAnswer.updated_at` Max via session |
| 시험 | `ExamAnswer.created_at` Max via session (insert만, update는 못 잡음) |

- 마지막 활동이 5분 이내 + 세션 `in_progress` → "활동 중"
- 동시에 여러 과목 있으면 가장 최근 활동 1개만 표시
- `current_activity_map()` 한 번 호출에 5개 서브쿼리 — `Max + filter` 로 학생 단위 집계

### 클라이언트 1초 틱
- 서버에서 `started_at` 의 epoch ms 를 `data-live-start` 속성으로 전달
- `setInterval(tick, 1000)` — 칩 안 `.live-time` 만 갱신, 페이지 새로고침 없이 카운트업
- 펄스 도트(`.live-dot`) CSS 애니메이션으로 라이브임을 강조

### 한계
- 시험은 답안 **수정** 시 타임스탬프가 안 잡힘 (`ExamAnswer.created_at` 만 있고 `updated_at` 없음). 정확도가 더 필요하면 1줄(`updated_at = models.DateTimeField(auto_now=True)`) + 마이그레이션이면 해결됨 — 향후 보강

**관련 파일:**
- `report/study.py` — `current_activity_map`, `fmt_hms`, `SUBJECT_LABEL`
- `report/views.py` — `study_board` (row.live 주입)
- `report/templates/report/study_board.html` — `.live-chip` CSS + 1초 틱 JS

## 5. 영작 이어풀기

학생이 20문제 중 4문제 풀고 나간 경우 → 다시 접속 시 5번부터 자동 재개.

### 자동 재개 흐름
1. **`start_session(unit_id)`** 가 같은 `(student, unit, start_index, end_index, view_mode)` 의 `finished_at__isnull=True` 세션을 찾아 redirect
2. **`session_view`** 가 기존 `WritingAttempt.is_correct=True` 를 `(problem_id, word_index) → correct_answer` 맵으로 복원
3. 각 단어 메타에 `auto=True, value=정답, resume=True` 셋팅 — 미리 채워서 잠금
4. `start_idx` = 첫 미완료 문제 인덱스 → JS 가 `loadProblem(START_IDX)` 부터 시작

### 핵심 디테일
- **resume 단어는 `autoSubmitWord` 스킵** — 이전 세션에서 이미 점수가 잡혔으므로 재크레딧/콤보 깨짐 방지
- CSS `.is-resume` 초록 톤으로 자동 채우기(회색 — 관사 등)와 구분
- 진입 시 `'📌 이어풀기 — 문제 N부터 시작'` 토스트로 안내

### 학생 체감
- 이탈 후 다시 "응시 시작" 클릭 = 풀던 자리로 자동 점프
- 이전 점수·콤보 보존, 신규 풀이만 가산
- 새 단원/새 범위로 가면 새 세션 생성 (동일 범위만 재개 매칭)

**관련 파일:**

## 6. 단어 스와이프 TEST

현황판 단어 칩 클릭 → 카드 한 장씩 좌우 스와이프 자기평가 → 종료 시 점수.

### 흐름
```
[현황판 칩 "내신단어TEST 1~100 미응시" 클릭]
       ↓
[/training/vocab/range/<rt_id>/swipe-menu/]   ← 2옵션 선택
       ↓
   ┌──────────────────────────────┐
   │ 🟣 전체 단어 TEST  (N단어)    │
   │ 🟡 오늘 별표만 TEST  (M단어)  │
   └──────────────────────────────┘
       ↓
[/training/vocab/range/<rt_id>/swipe/?src=all|star]
- 카드: 영어 단어 + 한글뜻 동시 노출 (자기평가)
- 오른쪽 스와이프 / → 키 / 알아요 버튼 = 정답
- 왼쪽 스와이프 / ← 키 / 모름 버튼 = 오답
       ↓
[종료 — 결과 보기]
- 일괄 제출 → VocabSession(MODE_TEST) + 일괄 VocabAttempt
- 오답 단어는 StudentWordStar 자동 추가
       ↓
[점수(0~100점) + 알아요/모름 개수]
```

### 옵션 의미
- **전체 단어 TEST**: 범위 내 단어에서 출제 — 기존 `select_test_words` (별표 70% + 어려운 단어 30%)
- **오늘 별표만 TEST**: `StudentWordStar.created_at__date=today` 인 단어 전체 — **단원 무관**, 그날 모은 모르는 단어 한 번에 응시
- 단어 0개인 옵션은 disabled 처리

### UI 디테일
- 카드 드래그 시 좌우 회전(`transform: translateX rotate`) + "알아요" / "모름" 워터마크 페이드 인
- 진행률·정답수·오답수 상단 실시간 표시
- 교사가 다른 학생 칩 클릭 → preview 모드 (노란 배너 + DB 저장 X, 점수만 계산)

### 함정 — `<script>` 안의 `{% csrf_token %}`
- `{% csrf_token %}` 출력은 `<input type="hidden">` 인데 `<script>` 안에 있으면 JS 파서가 깨짐 → `const WORDS = [...]` 가 정의되기도 전에 중단 → 빈 페이지
- 해결: `<form style="display:none;">{% csrf_token %}</form>` 으로 script 바깥 hidden form 으로 이동
- JS 는 `document.querySelector('[name=csrfmiddlewaretoken]')?.value` 로 읽음
