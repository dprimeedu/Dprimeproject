## 3. 영작훈련 시스템 (핵심 신규 작업)

### 3.1 새 Django 앱: `writing/`

**8개 모델**:
- `WritingUnit` — 단원
- `WritingProblem` — 문제 (한글/영어/단어별한글뜻 JSON)
- `UnitAssignment` — 선생님 → 학생 단원 배정
- `WritingSession` — 학생 1회 풀이 세션
- `WritingAttempt` — 단어 단위 시도 로그 (9컬럼)
- `StudentProfile` — 게임화 (XP/레벨/콤보)
- `Achievement` — 배지 정의 (13개 시드 등록 완료)
- `StudentAchievement` — 학생-배지 매핑

### 3.2 핵심 기능 구현 완료

**선생님 화면**
- `/training/writing/admin/upload/` — 엑셀 업로드 (단원명/학년/출판사 + 파일)
- `/training/writing/admin/units/` — 단원 목록
- `/training/writing/admin/units/<id>/` — 단원 상세 + AI 한글뜻 생성

**학생 화면**
- `/training/writing/` — 학생 홈 (배정 단원 + 프로필 카드)
- `/training/writing/unit/<id>/start/` — 세션 시작
- `/training/writing/session/<id>/` — 풀이 화면 (★ 핵심 UI)
- `/training/writing/result/<id>/` — 결과 화면 (점수/배지/통계)

**AJAX API** (3개)
- `POST /api/check-word/` — 단어 채점 + 게임 로직
- `POST /api/complete-problem/` — 문장 완료 처리 (perfect sentence 보너스)
- `POST /api/complete-session/` — 세션 완료

**관리자 페이지** (Django admin)
- 8개 모델 전부 등록됨

### 3.3 게임 메커닉

**점수 시스템**
- 1차 정답: +10점
- 1번 힌트(한글뜻) 후 정답: +6점
- 2번 힌트(첫글자) 후 정답: +3점
- 3번 정답 공개: +1점 (학습 인정)
- Speed Bonus (3초 안): +2점
- Perfect Sentence (한 문장 무실수): +20점

**콤보 보너스**
- 단어 콤보 5/10/20/50: +10/+20/+40/+100
- 문장 콤보 3/5/10: +50/+100/+200

**레벨/호칭** (XP = 누적 점수, 100 XP/레벨)
- Lv 1-2 새내기 / Lv 3-5 견습생 / Lv 6-10 영작러 / Lv 11-20 영작러+ / Lv 21-50 영작마스터 / Lv 51+ 영작신

**배지 13개** (자동 부여)
- 🌱 첫걸음 / ⭐ PERFECT TEN / 💯 PERFECT 100
- 🔥 5콤보 / 10콤보 / 👑 콤보 마스터
- ✨ Perfect Sentence / 🏆 Perfect Unit
- 📅 일주일 개근 / 한달 개근
- ⚡ 스피드 데몬 / 🌅 얼리버드 / 🦉 올빼미

### 3.4 AI 통합 (Gemini)

- **모델**: `gemini-2.5-flash-lite` (가장 빠름)
- **방식**: batch 처리 (20문제 1회 API 호출)
- **백그라운드**: threading + 2초 폴링으로 진행도 표시
- **속도**: 295문제 약 40초 (이전 20분 → 30배 단축)
- **비용**: 무료 한도 내 (단원당 0~10원)
- **결과**: 영어 단어별 문맥상 한국어 뜻 자동 생성

예시:
```
"In under three years was the project finished"
→ In:(시간)~에 / under:채 안 / three:셋 / years:년
   / was:됐다 / the:(관사) / project:프로젝트 / finished:끝난
```

### 3.5 보안

- ✅ 정답은 클라이언트에 절대 안 보냄
- ✅ 모든 채점은 서버 측 AJAX
- ✅ CSRF 토큰 검증
- ✅ 시도 횟수 서버 검증 (조작 방지)
- ✅ 로그인 사용자만 풀이 가능

## 6. ⚠️ 보안 패치 (협업자와 합의 필요 — 미진행)

운영 NAS 사이트에 그대로 있는 보안 이슈 3가지:

| # | 위치 | 문제 | 영향 |
|---|---|---|---|
| 1 | `config/settings.py:25` | SECRET_KEY 평문 git commit | 세션/토큰 위조 가능 |
| 2 | `config/settings.py:28` | `DEBUG=True` | 운영 에러 시 정보 노출 |
| 3 | `config/settings.py:30` | `ALLOWED_HOSTS=['*']` | 호스트 헤더 변조 공격 |

**제안 해결**: `.env`로 분리 (지금 우리 로컬엔 .env 있지만 settings.py가 아직 그걸 안 읽음)

**왜 안 했는가**: 운영 NAS 반영 시 NAS에도 `.env` 만들어야 하고 키 교체 시 기존 세션 다 끊김. 협업자와 시점 협의 후 진행하기로 보류.
