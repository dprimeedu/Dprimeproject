## A. 시험 빨파 — 3단계 워크플로

### 새 ExamSession 상태 (migration 0010, 0011)
| 필드 | 의미 |
|---|---|
| `redblue_released` (기존) | 선생님이 빨파 정답 공개 누름 |
| `student_redblue_done` (신규) | 학생이 빨파답 보고 자기채점 완료 |
| `teacher_final_confirmed` (신규) | 선생님 최종 확인 — 이 시점에 학생 홈에서 사라짐 |
| `ExamAnswer.review_marked` (신규) | 학생이 2차에서 '모르는 문제' 표시 — 답이 맞아도 오답처럼 표시 |

### 학생 흐름
1. 1차 자동채점 → 2차(틀린문제 빨파 재시험) 제출
2. **선생님 [공개]** → 학생 홈에 이전 시험 카드 그대로 + **다음 회차 자동 배정**(부수효과)
3. 학생: 빨파답 보고 자기채점 → **[빨파채점 완료]** 큰 초록 버튼
4. **선생님 [최종 확인]** → 이전 시험이 학생 홈에서 사라짐

### 2차 '모르는 문제' 토글
- `r2-cell` 우상단 둥근 [? 모르는 문제] 버튼 — 답 입력과 독립
- 노란 강조 + 카드 배경 옅은 노랑
- 제출 후 ★ 노란 배지로 오답 표·빨파 정답 화면에 함께 노출

### 자동 다음 회차 배정
- `find_next_mock_paper` (회차): 같은 학년의 다음 (연도, 강)
- `find_next_mock_type_paper` (유형별): 같은 (학년·카테고리·태그)에서 (range_end+1) ~ (range_end+size)
- `find_next_paper` 디스패처 — `release_redblue` 가 호출
- backfill 한 번: 이미 빨파공개됐던 mock_type 케이스에도 누락분 ExamAssignment 생성

### 학습 현황판 (`/training/study/`)
- 컬럼 '시험' → **'정답입력'** 으로 통합 (SUBJECT_META 라벨 변경, 라이브 칩/리포트 일관)
- 정답입력 cell 4단계 상태 색 분리: **공개대기(노랑) → 학생 채점 중(회색) → 최종확인 대기(파랑) → 완료(초록)**
- `can_release` 인라인 🔓 / `can_finalize` 인라인 ✅ 버튼 (AJAX, 공통 `doAjax` 헬퍼)
- 학생 ExamAssignment 만 있고 ExamSession 없는 '미응시 배정' 도 회색 배지로 노출
- 내신은 한 세트 부분입력 패턴이라 채점완료 세션을 날짜 무관 항상 표시
- 학생 개인 낱말카드(`WordCard` word·meaning 모두 채워진 것) 수를 단어 cell 에 **개인 단어 N개** 초록 배지로 — 클릭 시 `vocab:student_cards` 진입

### 학생 홈 (`/training/exam/`)
- `my_sessions` 필터 `redblue_released` → `teacher_final_confirmed` (공개만 된 상태에선 카드 유지)
- 빨파 표시 폰트: 본문은 **나눔스퀘어라운드**, [빨]/[파] 지시문은 고딕 계열
- `static/fonts/NanumSquareRoundR.ttf` 를 `@font-face` 로 로드, fallback 맑은고딕/Noto Sans KR

## B. 모의고사 자료 페이지 (`/academy_list/`)

회차 멀티 체크 → '선택 항목 열기' 다중 탭 흐름에서, **한 화면 통합 선택 + 직접 다운로드** 로 전환.

### 최종 배치 (위→아래)
1. 검색
2. 학년/년도/월 **드롭다운** (single-select, '선택' placeholder)
3. **번호 그리드** (18~50 고정) + **직접 입력** 통합
   - 회차에 실제 있는 번호: 보라 + **기본 on(선택 상태)**
   - 없는 번호: 점선 dim
4. **변형문제 유형** chip multi-select — 기본 전체 on
5. 선택된 번호 누적 (× 개별 제거 / 모두 지우기)
6. **PDF 출력** / **HWPX 출력** 두 버튼 (빨강·파랑)

### 사용자 피드백을 따라 5~6회 갈아엎음
- 칩 ↔ 드롭다운 왔다 갔다 (최종 드롭다운)
- '번호 추가입력' → '직접 입력' 으로 라벨 (위치는 번호 그리드 안)
- 변형 유형 헤더의 '(여러 개 선택 가능 · 기본 전체)' 안내 문구 제거
- 진입 시 **고1 + 최신 연도/월** 자동 선택 → 빈 화면 없이 바로 번호·유형까지 노출

### 데이터·매핑 보정
- KEY_TABLE 의 비정규 `year` 값('24', '23') 26행 + 관련 CountTable 101행 삭제 (정상 '2024'/'2023' 별도 존재)
- 2026 6월 고1/고2 `total_number` 가 한글 형식(`'고1 2026년 6월 20번'`) 36건 → `YYYY-MM-NN` 으로 정규화 → 번호 그리드 정상 노출

### round_numbers 이중 인코딩 버그
- view 에서 `json.dumps()` 한 결과를 또 템플릿의 `json_script` 필터에 넘겨 이중 직렬화 → JS `ROUND_NUMBERS` 가 문자열이 되어 `[key]` 가 undefined
- 수정: view 는 dict 그대로 넘기고 `json_script` 가 단일 직렬화

### nginx /media/ 라우팅
- nginx 컨테이너에 `/media` 호스트 마운트가 없는 상태에서 `alias /media/` 가 빈 디렉터리를 가리켜 모든 업로드 이미지 404
- `nginx/default.conf` 의 `location /media/` 를 `proxy_pass http://django:8000` 으로 변경
- `config/urls.py` 가 production 에서도 `django.views.static.serve` 로 `/media/` 처리하도록 이미 되어 있어 안전

### 빨파 지문 데이터 정비 (다른 작업과는 무관하지만 같은 날)
- 빨파 ExamQuestion.text 의 passage 부분을 QuestionData.지문(문제출력 D열) 으로 교체 — 답이 보이는 원문 대신 빈칸·선택지 마커가 있는 문제 버전
- 1차 backfill 4,432문항 교체 / 이미 동일 2,529 / 장문 follower(842 → leader 지문 복사) 808 추가
- `[빨]`/`[파]` 지시문 라인은 보존

## C. 변형문제 데이터 일괄 import + 텍스트 정제

`Modified_Questions` 총 4,182 → **56,955건** (+52,773).

### 회차별 import 흐름
1. `/redblue_root/모의고사/{학년}/{년}년/{년}년 {월}월/문서자동화/{년}년 {학년} {MM}월 모의고사 통합.xlsx`
   (옛 폴더는 `{월}월 모의고사` 패턴도 fallback)
2. `변형문제` 시트 — 열: 색인·문제·유형·지문·보기·정답·변형·강·번호·**단원**·분류번호
3. **단원** 컬럼(`YYYY-MM-NN` 또는 `YY-M-NN`) → KEY_TABLE.total_number 매칭 → pk_number 조회 → INSERT
4. 옛 시트의 `검색` 컬럼이 단원 역할인 경우도 fallback
5. 2자리 연도(`24-11-18`) → 4자리(`2024-11-18`) 정규화
6. 같은 색인(Index) 중복 INSERT 방지

### 일괄 import 결과
- 2026년 3월: 고1/2/3 → 971건
- 2024년 (고1/2 전부): 4,257건
- 2018~2023 (고1/2/3): 34,641건
- 2025년 (고1/2 전부 + 고3 3/4/6): 12,033건

### 텍스트 정제 (PDF·HWPX 양쪽)
- Excel OOXML `_x000D_` (CR) / `_x000A_` (LF) → 진짜 줄바꿈
- literal escape `\r\n` (4글자) + 실제 컨트롤 모두 줄바꿈으로 통합
- `\t` → 공백 (한/글 렌더링 오류 회피)
- 연속 빈줄 3+ → 2개로 축약
- 보기 분리: 줄바꿈이 없거나 한 줄에 ①②③ 마커가 같이 있으면 **마커 앞에서 추가 분리**
- uFFF0 강조 마커: PDF 는 `<span class="mark">` 밑줄+굵게, HWPX 는 빌더가 별도 처리

### 변형 유형 동의어 통일
- `연결사` → **`연결어`** / `지칭추론` → **`지칭대상`** (UI 칩 목록 통합 + 필터에서 DB 옛 표기도 매칭)
- `_expand_type_filter` 헬퍼로 두 download 뷰 공통 사용 — `'순서,연결어'` → `['[순서]', '[연결어]', '[연결사]']` 같이 확장

## D. 모의고사 PDF — ReportLab → WeasyPrint(HTML→PDF) 전환

ReportLab Platypus 로 2단/박스/폰트 다 흉내내봤지만 한/글 인쇄 양식과 결국 차이 남.
**HTML 템플릿 + CSS column-count** 로 갈아엎으니 nernter.com 레퍼런스와 거의 같은 결과.

### 컨테이너 사전 작업
```
apt-get install libpango-1.0-0 libpangoft2-1.0-0 libcairo2 libgdk-pixbuf-2.0-0
                 fonts-nanum fonts-nanum-extra
pip install weasyprint (requirements 68.1)
```

### 템플릿 `academy/print_paper.html`
- `@page` top-center 머리말 (첫 페이지만 비움) + 페이지번호 footer
- 본문: `column-count: 2; column-gap: 6mm; column-fill: auto;`
- 지문 회색 박스: `#fafafa` 배경 + `#d4d4d8` 0.4pt 테두리
- 보기: ①②③④⑤ hanging indent (`text-indent: -6mm; padding-left: 6mm;`)
- 정답: `page-break-before: always; column-count: 1;` 새 페이지 단단
- 폰트: **Nanum Gothic** + Malgun Gothic / 맑은 고딕 / Arial fallback

### download_pdf 뷰
- `render_to_string('academy/print_paper.html', {...})` → `HTML(string=...).write_pdf()`
- 기존 ReportLab 함수는 `_download_pdf_OLD_reportlab` 으로 이름 변경 후 보존

## 운영 메모

- **로컬→NAS 데이터 sync 부재 확인**: 사용자 측 업로드는 KEY_TABLE 회차 ID 만 갱신하고 question_data / Modified_Questions 본문은 별도 경로(=교재폴더 마운트의 통합 xlsx)에서 import 해야 됨. 폴더 구조 패턴이 연도별로 다양(`{년}년 {월}월` / `{월}월 모의고사` / 옛/신 헤더 등) 이라 import 스크립트가 보정 로직을 다 들고 있어야 함.
- 고3 2025 7/9/10/11월 + 일부 옛 회차는 통합 엑셀의 **변형문제 시트 자체가 비어 있어** (3870행이지만 색인 None) import 불가. 엑셀 채워지면 동일 스크립트로 재import 가능.
