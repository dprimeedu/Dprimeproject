## 0. 결론 (중요)
- **유형 콘텐츠도 import 불필요였습니다.** 웹 `QuestionData`(270회차·7,771문항)에 **문항별 `유형` 필드가 이미 완비**돼 있습니다(7,771/7,771: `[순서][빈칸][문장넣기][무관한문장][어휘][밑줄의미][어법]…`). 빨파 ExamQuestion에도 동일 유형 + `ref_number=색인(YYYY-MM-NN)` 존재.
- **고3 = 3,660행**으로 권용국 측 내신빨파 DB(고3 3,660)와 **모집단 정확히 일치**.
- 따라서 유형별 시험지 = **QuestionData를 (학년+유형태그) 필터 → 색인순 재번호 → 범위 슬라이스**. 권용국 모고엑셀답지생성.py와 같은 규칙.
- **검증**: `고3 / [순서][문장넣기][무관한문장] / 257-288` → **32문항**, #257 = `2017-11-37 [순서] 정답1` … 클라이언트 재번호와 일치 확인.

## 1. 신설 엔드포인트 (확정·배포·검증됨)

```
POST https://dprimeedu.synology.me/training/exam/api/import-student-mock/
헤더: X-Exam-Token: pedu-exam-2026   (또는 body "token")
Content-Type: application/json; charset=utf-8
```

### 요청 body — items 배열, kind별
**회차(round):**
```json
{"name":"양지후", "kind":"round", "exam_key":"2013-고3-11", "goal":23}
```
- `exam_key` = `YYYY-고N-M` (월 M = 강: 3/6/9/11 …).

**유형(type):**
```json
{"name":"양지후", "kind":"type", "grade":"고3",
 "type_name":"순서문장넣기무관한",
 "types":["[순서]","[문장넣기]","[무관한문장]"],
 "start":257, "end":288, "goal":32}
```
- `grade` = `고1`|`고2`|`고3`
- `type_name` = 합본 교재명(라벨, 화면 표시·식별용)
- `types` = **권용국이 COMBO_TYPES로 해석한 태그 리스트**(웹은 이 태그로 QuestionData 필터). ← 웹이 COMBO_TYPES 안 가져도 됨. 태그 문자열은 `[순서]`처럼 대괄호 포함 그대로.
- `start`/`end` = 색인순 재번호의 슬라이스(예 257-288).
- `goal`(선택) = 학생별 하루목표 → `ExamAssignment.daily_goal`(홈 카드 "오늘 목표 N문항").

공통: `name` = Member.username. 동명이인/미존재 skip.

### 응답
```json
{"success":true, "assigned":N, "updated_goal":N, "skipped_dup":N,
 "not_found":N, "no_content":N, "bad_key":N, "results":[...]}
```
- `results[].status` = `assigned` | `assigned_no_content` | `not_found` | `dup_name` | `bad_exam_key`(round 키 형식) | `bad_type_params`(type 필수값 누락).

### 검증 결과(2026-06-16 실제 HTTP, 운영)
- round `정예진 + 2011-고3-9` → `assigned`(33문항, goal 23).
- type `정예진 + 고3 순서문장넣기무관한 257-288` → `assigned`(**32문항**, #257=2017-11-37[순서]). (테스트 배정은 정리함)

## 2. 합의 필요 사항(명세 §7) — 최종 답변
1. **콘텐츠 원천?** → 회차·유형 **모두 웹 QuestionData에 이미 있음**. import 불필요. (유형 = 유형태그 필터, 회차 = 학년·연도·강)
2. **exam_key 규약?** → 회차 `YYYY-고N-M`. 유형은 exam_key 대신 **(grade, type_name, types[], start, end)** 구조 전달(유형책은 단일회차가 아니라 재번호 가상책이므로).
3. **import-naesin 확장 vs 신설?** → 배정 전용 **신설**(`import-student-mock`). 콘텐츠 import 없음.
4. **응시 UI 내신 재사용?** → 회차·유형 **모두 기존 OMR 응시화면 재사용**(1~5 입력→즉시 자동채점→오답 2차→교사확인). 배정되면 학생 홈에 카드로 노출, '응시 시작'.
5. **배정 단위?** → **파싱된 구조**(위 payload). 서버는 슬롯 원문 재파싱 안 함.

### 재번호 정렬키(중요·일치 보장)
웹 정렬 = `QuestionData.order_by('연도','강','번호')` = 색인(YYYY-MM-NN) 오름차순. zero-pad라 문자열 정렬과 동일. **권용국 색인 정렬과 일치**(검증됨). 단, 유형 모집단이 바뀌면(문항 추가/삭제) 양쪽 재번호가 같이 흔들리므로, DB 갱신은 양쪽 동기 필요.

## 3. 권용국(클라이언트) 측 — 지금 바로 가능 (명세 §6)
1. `개별단어장생성.py` 슬롯 파서로 AP/AR에서 **round/type 슬롯** 추출.
   - round → `{name, kind:'round', exam_key:'YYYY-고N-M', goal}`
   - type → `{name, kind:'type', grade, type_name, types:COMBO_TYPES[합본명], start, end, goal}`
2. `upload_mock_assignment()` 추가 → 위 §1 엔드포인트 POST(토큰 동일, `upload_naesin_range` 패턴).
3. ⑤ 실행 시 자동 호출 배선.

> **유형책 → types 변환만 클라에서** 해주시면 끝(웹은 태그로 바로 필터). COMBO_TYPES를 웹에 중복 둘 필요 없음.
