> 작성 2026-06-06. 구글 드라이브 + Apps Script + 로컬 엑셀 + 카톡 GUI 자동화로 흩어진 학생관리 흐름을 운영 중인 Django 홈페이지(`dprimeedu.synology.me`)로 통합.

## 0. 결정사항 (2026-06-06 합의)
- **웹 이전 범위**: ② 학생 입력 ~ ⑤ 일일 리포트 생성 (전체).
- **카카오 전송**: **하이브리드(A)** — 서버가 리포트 생성, 회사 PC의 작은 프로그램이 받아 기존 방식으로 카톡 전송. (단톡방 자동 전송은 카카오 공식 API로 불가하므로 데스크톱 자동화는 회사 PC에 유지.)
- 작업 방식: 집 노트북에서 코드 작성 → SynologyDrive 동기화 → 회사에서 실행/배포. (실행·검증은 회사. 참고 [[home-laptop-workflow]])

## 3. 기존 Django 자산 (재사용)
- `member.Member` — 커스텀 User. `member_type`(user/academy_admin/admin), `login_id`, `is_approved`, `phone`. **학교/학년 필드 없음 → 추가 필요.** 교사 역할 없음 → 필요 시 확장.
- `academy.Academy` — 학원 단위. `course.Enrollment` — 학생↔코스 등록. **반(class) 개념 없음 → 추가 필요.**
- `summary`(요약문완성), `vocab`(단어훈련), `writing`(영작훈련) — **이미 학생이 웹에서 보는 시험 앱**. 세션·채점 모델 존재(`SummarySession`/`SummaryBlankAnswer` 등). → 시험 결과 일부를 리포트에 **자동 연동** 가능.
- `academy_tools` — 모의고사/EBS/교과서 문제 내보내기. 모의고사·내신 데이터는 이미 DB sync됨.
- 컨벤션: `db_table` 앱 프리픽스, `verbose_name` 한글, FK→`settings.AUTH_USER_MODEL`+related_name, status choices, `percent` property 등 (`summary/models.py` 따름).

## 4. 새 앱 설계 — `report` (학습관리·데일리리포트)
> 앱명 확정 전(`report`/`care`/`lms` 후보). 1차는 학생관리표를 거의 그대로 미러링하는 단일 일일기록 모델로 시작 → 추후 정규화.

### 4.1 모델
- **ClassRoom(반)** — `academy`(FK), `name`, `weekday_time`(요일시간), `teacher`(FK Member, nullable). 학원 내 반 구분.
- **StudentProfile 확장** — `member.Member` 또는 별도 1:1: `school`(학교), `grade`(학년), `class_room`(FK), `chatroom_name`(카톡 단톡방명 = 리포트 파일명 규칙 `이름(학교) 프라임에듀 단톡방`).
- **DailyRecord(일일 학습 기록)** — 핵심. unique(student, date). 학생관리표 51컬럼을 필드로 매핑:
  - 기본: `student`, `date`, `weekday_time`, `attendance`(출석/지각/결석/미입력)
  - 시험결과(텍스트+해석): `grammar_summary_result`(어법/요약문), `vocab_result`(단어시험), `writing_result`(영작), `reading_result`(TEST)
  - 과제: `hw1_rate`, `hw2_rate`, `hw_rank`, `reading_hw1_result`, `reading_hw2_result`, `grammar_no`, `grammar_hw_result`
  - 오늘/다음 과제: `today_reading1`(+link1), `today_reading2`(+link2), `today_grammar`, `next_reading1`, `next_reading2`, `next_grammar`
  - 범위: 단어장/단어·영작·독해 각 오늘·숙제 시작/끝 (range 필드들)
  - `teacher_comment`(선생님 한마디)
  - 메타: `created_by`, `created_at`, `report_image`(생성된 PNG), `report_generated_at`
- **(선택) 정규화 버전**: 시험은 별도 `ExamResult`, 출결은 `Attendance`로 분리 가능. 1차는 보류.

### 4.2 자동연동 vs 수동입력
| 리포트 필드 | 출처 | 방식 |
|---|---|---|
| 단어시험결과 | `vocab` 세션 | 자동(해당 날짜 세션 점수) + 교사 override |
| 영작시험결과 | `writing` 세션 | 자동 + override |
| 어법/요약문결과 | `summary` 세션 | 자동 + override |
| 출석, 과제1·2달성률, 독해/문법 과제결과, 다음과제, 한마디 | 교사 입력 | 수동(웹 폼) |
> ②"학생 입력"의 정확한 항목·현행 Apps Script 채점 규칙 확인 필요(§7).

## 6. 리포트 생성 (⑤ 이식)
- `학습보고서.py`의 `create_and_save_report_1()` 로직을 서버 유틸로 이식. matplotlib **Agg(headless) 백엔드**로 서버에서 동일 PNG 생성. 한글 폰트(Malgun Gothic 등) Docker 이미지에 포함 필요.
- 출력: `DailyRecord.report_image`(media) 저장. 파일명 규칙 `이름(학교) 프라임에듀 단톡방.png` 유지(카톡 대화방 검색 매칭).
- 대안: matplotlib 대신 HTML/CSS 리포트 → 웹 열람 + (필요시)이미지 변환. 1차는 기존 PNG 유지로 리스크 최소화.
- 트리거: 교사가 그날 입력 완료 후 "리포트 생성" 버튼 / 또는 일 1회 배치.

## 7. 하이브리드 카카오 전송 (⑥ 유지)
- **서버 API**: `GET /report/api/daily/?date=YYYY-MM-DD` → 토큰 인증, 그날 생성된 리포트 목록 `[{chatroom_name, png_url}]` 반환.
- **회사 PC 페처(신규 소형 스크립트)**: API 호출 → PNG들을 `Desktop\카카오톡`에 내려받음 → 기존 `Kakaosend.py` 그대로 실행해 단톡방 전송.
  - 즉 `학습보고서.py`(로컬 생성)만 "서버에서 내려받기"로 대체. `Kakaosend.py`는 무수정 유지.
- 향후 옵션: 카카오 알림톡(사업자 채널, 템플릿 승인·유료)으로 "리포트 링크" 발송 전환 검토.

## 9. 현행 채점 로직 분석 (Apps Script → Django)
> 원본: 구글 Apps Script(학생 통합관리 메뉴). 학생별로 **개인 구글 스프레드시트 1개**가 폴더(`FOLDER_ID=1B6o…hOuM`)에 있고, 그 안 시트별로 채점. 점수는 마스터(학생관리표) 행에 자동 기록.

### 채점 유형
| 메뉴 | 시트 | 입력→정답 | 점수 | 마스터 기록열 | 부가 |
|---|---|---|---|---|---|
| 단어 통합 | `단어` | C(입력)==H(정답), 3~42행 | 정답/문항 | 14(단어시험결과) | 한글오답·영영오답 추출, 5개↑ 틀리면 재시험 초기화 |
| 어법 통합 | `어법` | G열 O/X | 정답/문항 | 7(어법/요약문) | E열 틀린횟수 누적+색상, 어법오답, 40문항↑ 셔플, 중등/고등 자동판별 |
| 내신어법 통합 | `내신어법` | G열 O/X | 정답/문항 | 21 | 위와 동일(내신어법오답) |
| 요약문완성 | `요약문완성` | H·I열 O/X(빈칸 ⓐⓑ) | 정답/빈칸 | 7 | 오답 빨강 표시, 전부정답 행 숨김 |
| 모고/유형별 | `모고`/`고3`등 | C(입력)==F(정답) **숫자비교**, 8행~ | 정답/응답 | 38·39(독해과제1·2결과)+35·36(과제1·2달성률) | B5=목표문항수→달성률, 빨파문제/빨파정답 오답저장, 시험명 매칭으로 과제1/2 자동분배 |
| Part1~4/내신TEST/내신객관식빈칸 | 동명 | A=번호,B=입력,I=정답 **숫자비교** | 정답/응답 | — | 오답시트, 풀이행 숨김 |

- **모의고사 답안 = 숫자 1~5 객관식** (가장 단순). 내신 객관식도 숫자 비교.
- 마스터 컬럼(1-based): 7=어법/요약, 14=단어, 21=영작/내신어법, 35·36=과제1·2달성률, 38·39=독해과제1·2결과, 42·44=오늘독해과제1·2.

### Django 이식 시 단순화 포인트 (큰 이득)
- 채점 본질 = **입력 vs 정답 비교 → O/X → 점수**. 서버에서 trivial.
- Apps Script 분량 대부분(오답시트 생성·셀 색칠·행 숨김·필터·셔플)은 **DB 모델 + 오답노트/재시험 기능**으로 대체되며 사라짐.
- `isDiffExam`/`normalizeName`(06월 vs 6월 문자열 매칭으로 과제1/2 분배) **꼼수 전부 불필요** — DB에선 시험/단원을 FK로 정확히 연결.
- "틀린횟수 누적+색상", "5개↑ 재시험 초기화", "40문항↑ 셔플" → 오답노트 모델의 정식 기능으로 재현(더 견고).

## 12. 구현 2차 — 내신 정답 웹 업로드 + 통합 시험 모델 (2026-06-06)
내신 시즌 대응. `exam` 앱을 **출처 통합형(모의고사+내신)** 으로 리팩터 후 내신 흐름 추가.

### 통합 모델 (exam/models.py 교체)
- `ExamPaper(source=mock|naesin, ...)`: 시험지 1장. mock=학년·연도·강(QuestionData 실시간), naesin=학교학년·시험명(season)·카테고리.
- `ExamQuestion(paper, number, answer, qtype)`: 내신 문항(정답 저장). 모의고사는 QuestionData 사용해 미저장.
- `ExamAssignment(paper, student)` / `ExamSession(paper, student, ...)` / `ExamAnswer(session, number, ...)`.
- 내신 카테고리 6종: **Part1~4 / 내신TEST / 내신객관식빈칸** (구 내신부교재/교과서 구분 폐기 → Part1~4로 통일).

### 내신 정답 업로드 경로 (구글 → 웹 전환)  ※ 2026-06-06 확정 방식
- **데이터 소스 = '정답' 파일**(구글에 올리던 그 파일): `교재폴더\내신\...\2026년 1학기 기말고사\{학교}\...{(partN)}\... 정답\... OO 정답.xlsx`.
  - 형식 A=색인(번호 1,2,3..), B=정답(1~5), C=유형. **행 순서 = 시험지 순서**(이미 선별·정렬·번호매김 완료 → '그대로' 읽으면 됨, 정렬 불필요).
  - 출력/순서 원리는 `완성(코드)\B. 부교재 출력.py`에 있음(엑셀 행 순서 = 출력 순서, 번호=색인).
- **카테고리 분류**: `변형문제 정답`→경로의 `(partN)`로 Part1~4 / `내신TEST 정답`→내신TEST / `객관식빈칸 정답`→내신객관식빈칸.
- **서술형 제외**: 유형에 '서술형' 포함 행은 전 카테고리 제외(자동채점 불가, 정답이 텍스트). 비1~5도 제외.
- **신규 푸셔** `...\TEST 자동화\내신웹업로드.py`: 시즌폴더(기본 `2026년 1학기 기말고사`) 하위 `*정답*.xlsx` 스캔 → 재원생 학교만 → 학교별 카테고리 묶어 import API POST(urllib). `PEDU_DRY=1`로 추출만 검증 가능.
  - ⚠ **통합본 직접 읽기는 폐기**(학교당 통합본이 백업·모의고사판·교재별 서브폴더로 여러 개라 정본 식별이 불안정). 정답 파일이 정본.
  - 드라이런 검증(2026-06-06): 9개 학교 정답파일 정상 추출(예: 백현고1 Part1 458/Part2 220, 청덕고3 Part1 721+객관식빈칸30). 단, 변형문제 문항수가 매우 큼(전체 변형세트) → 응시 UX(분할) 추후 조정 필요.
- **import API** `POST /training/exam/api/import-naesin/` (토큰 `X-Exam-Token`=settings.EXAM_IMPORT_TOKEN, 기본 'pedu-exam-2026'). body: {school_grade, season, categories:{Part1:[[번호,정답,유형],...], ...}} → ExamPaper upsert + ExamQuestion 전체 교체.
- ⚠ **정답은 코드 작성만으론 안 올라감 — `내신웹업로드.py` 실행 시 업로드**됨.

### ⚠ 회사에서 할 일 (순서)
1. `makemigrations exam` → `migrate` (ExamPaper/ExamQuestion 추가, ExamSession/Answer 구조 변경). ※ exam 첫 배포면 그냥 새로 생성됨.
2. 배포(git push→NAS). `EXAM_IMPORT_TOKEN` 환경변수 운영서버에 설정(미설정 시 기본값 사용).
3. 정답 업로드: `cd ...TEST 자동화` → `python 내신웹업로드.py` → 시험명 입력 → POST. (학교 한정 시 `PEDU_NAESHIN_SCHOOL_FILTER`)
