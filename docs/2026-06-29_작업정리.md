### ★ 비자명 함정 (다음 라운드에 또 만남)
- **`\n`=줄바꿈, `\n\n`=빈 줄.** `_compact_paragraphs`가 단일 `\n`을 공백으로 합침
  (단락은 `\n\n`만 유지). 순서 (A)(B)(C)·대화문·실용문·요약 구분은 `\n\n` 또는
  `_compact_paragraphs`의 KEEP 보호로 살려야 함. (`_esc`는 `\n`을 그대로 두고 한/글이 렌더)
- **None-판정 게이트보다 '표시용 정리'를 뒤에 둘 것.** `_pad_blank_labels`·
  `_strip_trailing_label_header`를 게이트 앞에 두면 `_is_broken_connector`(공백 생기면 오탐)·
  `_is_incomplete_ending`(헤더 떼면 요약구절 미완성 오판)이 멀쩡한 문항을 대량 제외함
  (검증 중 31건 잘못 제외 발견 → 게이트 뒤로 옮겨 0건).
- 본문 끝 `[3점]`이 `_strip_bracket_garbage`의 ']로 끝나는 줄 통삭제 룰에 걸려 지문이 통째로
  사라졌었음 → `_strip_score_marker`로 먼저 제거.

## 2. 보류한 결함 (정규화로 안전 자동수정 어려움 — 소스 데이터 단위 수정 필요)

멀쩡한 문항 훼손 위험이 있어 손대지 않음. 필요 시 엑셀 원본에서 직접 고치는 게 맞음.
- **제시문 뒤 단어 1개 잔재**(예 "…participants. Why") — legit 2문장 제시문과 구분 불가.
- **문장넣기 제시문 2개 / 본문 중간삽입 / 문장분리 오류** — 구조 corruption, 자동판별 위험.
- **(A)~(E) 밑줄 누락**(밑줄형인데 ￰ 마커 없음) — 밑줄 위치 데이터가 없어 복구 불가.
- **어휘/빈칸 (B) 라벨 누락 + 선지 미분리** — (B) 위치 추정 불가(`_extract_abc_options`가 A·B·C 다 있어야 분리).
- **지문에 관계없는 괄호**(예 "( as a species )", "( fight )") — legit 곁다리 괄호와 구분 위험.
