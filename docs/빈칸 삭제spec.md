# 빈칸 추론 문제 자동 필터링 작업 명세 (spec.md)

## 목적
영어 빈칸 추론 문제(수능/모의고사 스타일) 약 10,000개 중에서, **선지가 서로 너무 비슷해서 지문만으로 정답과 2위 오답을 변별할 수 없는 "애매한 결함 문제"**를 자동으로 찾아내 제외(flag)한다.

핵심 원칙: **형식이 비슷한 것 자체는 결함이 아니다.** 진짜 결함은 "지문 안에 정답을 지지하면서 동시에 2위 오답을 배제하는 문장(결정타 문장)이 존재하지 않는" 경우다.

---

## 입력 데이터
- 위치: (지정 필요 — 예: `./data/questions.json` 또는 `/mnt/user-data/uploads/`)
- 각 문제는 다음 필드를 가짐(없으면 분기 처리):
  - `id`: 문제 고유 ID
  - `passage`: 빈칸이 포함된 지문 (빈칸은 `__________` 표기)
  - `choices`: 선지 5개 배열 (①~⑤)
  - `answer`: **공식 정답 라벨** (있으면 ground truth로 고정, 없으면 LLM이 추론하되 신뢰도 낮게 처리)

> ⚠️ 데이터가 PDF/이미지면 OCR 선행 필요. 텍스트(JSON/CSV/표)면 바로 진행.

---

## 처리 파이프라인 (3층 구조)

### 1층 — 룰 기반 사전 필터 (빠름, 후보 선별용)
문자열·패턴만으로 판정. **여기서 걸린 건 "제외"가 아니라 "2층 검토 후보"일 뿐이다.**

다음 중 하나라도 해당하면 2층 후보로 표시:
- (a) 선지 5개 중 3개 이상이 동일/유사 **도입구**로 시작 (첫 5~7단어 정규화 후 일치율 ≥ 임계값, 기본 0.8)
- (b) 선지 5개 중 4개 이상이 동일 **꼬리구**로 끝남
- (c) 선지 쌍 간 평균 토큰 중복률(Jaccard) ≥ 임계값 (기본 0.5)
- (d) 선지 길이(토큰 수) 표준편차가 비정상적으로 작음 (선지가 다 비슷한 길이)
- (e) **글자수 기준: 선지 5개 중 3개 이상이 공백 포함 100자 이상**
  - 글자수는 `len(choice)` (공백 포함, 영문/숫자/기호 모두 1자로 카운트)
  - 임계값: 선지 1개당 100자, 문제당 3개 이상
  - 근거: 긴 선지는 "맞는 절 + 살짝 비튼 절"을 한 문장에 담아 변별을 흐리는 결함이 몰리는 경향. 단, 길다는 것 자체는 제외 사유가 아니며 2층에서 최종 판정.
  - 참고 측정값(이미지002.png 예시): ①106 ②110 ③99 ④102 ⑤121 → LONG 4개 → 플래그됨

출력: 각 문제에 `layer1_flagged: true/false`와 걸린 규칙 목록.

> 1층을 통과(=형식이 안 닮음)한 문제는 변별 문제가 거의 없으므로 기본 "유지". 비용 여유가 있으면 전수를 2층에 넣어도 됨(설정 옵션).

### 2층 — LLM 판정 (1층 후보만, 핵심)
1층에서 걸린 문제만 LLM에 전달. 지문+선지+공식정답을 주고 아래를 **구조화 JSON**으로 강제 출력:

1. **정답 고정**: `answer`가 입력에 있으면 그대로 정답으로 사용. LLM이 임의로 바꾸지 못하게 함. (없을 때만 추론하고 `answer_inferred: true` 표시)
2. **2위 오답 지목**: 정답을 제외한 4개 중 가장 매력적인 오답 1개를 LLM이 선택.
3. **결정타 문장 추출**: "정답을 지지하면서 **동시에** 2위 오답을 배제하는" 문장을 **지문 원문에서 그대로 인용**. 변형·요약 금지, 원문 그대로.
4. **결정타 유효성 판정**:
   - 인용한 문장이 정답을 지지하고 2위 오답을 실제로 배제하면 → `decisive_valid: true`
   - 문장을 못 찾거나 / 그 문장이 2위 오답도 똑같이 지지해버리면 → `decisive_valid: false`
5. **정답 자체 결함 체크**: 정답 선지가 지문과 모순되거나, 번역투 비문이거나, 지문에 근거 없는 내용이면 `answer_defect: true`.

**제외 판정 규칙**:
- `decisive_valid == false` → **제외(remove)**
- `answer_defect == true` → **제외(remove)**
- 둘 다 아니면 → **유지(keep)** (어려워도 변별되는 좋은 문제)

### 3층 — 사람 최종 확인 (선택)
2층에서 LLM이 confidence를 낮게 준 건(`confidence < 임계값`) 사람이 최종 확인하도록 별도 큐로 분리.

---

## LLM 출력 JSON 스키마 (2층)
```json
{
  "id": "2025-11-21",
  "answer": "①",
  "answer_inferred": false,
  "second_best": "⑤",
  "decisive_sentence": "지문에서 그대로 인용한 한 문장 또는 null",
  "decisive_valid": false,
  "answer_defect": false,
  "verdict": "remove",
  "confidence": 0.0,
  "reason": "①(직관적 길잡이)을 지지하면서 ⑤(웰빙 가치)를 배제하는 문장이 지문에 없음. 오히려 알코올·식단 근거는 ⑤도 지지함."
}
```

## verdict 값
- `keep`: 유지
- `remove`: 제외
- `review`: 사람 확인 필요

---

## 검증 케이스 (필터가 올바른지 확인용)
아래 5문제로 파이프라인을 먼저 테스트한 뒤 전체에 적용한다. 기대 결과와 일치해야 함:

| 문제 | 정답 | 기대 verdict | 이유 |
|---|---|---|---|
| 궁전/유리 (palaces, glass) | ③ | keep | 마지막 문장이 정답을 직접 지지 |
| 유교/가족 (Confucian, family) | ⑤ | keep | "가족은 본질적 가치" 문장이 ⑤ 지지 |
| 침묵의 나선 (spiral of silence) | ② | keep | "한쪽 주목/한쪽 침묵"이 ②의 양방향 구조 지지 |
| 야생동물 관리 (wildlife, inactive) | ② | keep | "active management 없는" 문장이 ②(비개입) 지지 |
| 미각/taster status | ① | **remove** | ①과 ⑤를 가르는 결정타 문장 없음 |

> 미각 문제가 `remove`로 나오고 나머지 4개가 `keep`으로 나오면 필터가 올바르게 작동하는 것.

---

## 산출물
1. `results.json` — 전체 문제별 판정 결과 (위 스키마)
2. `removed.json` — verdict가 remove인 문제만
3. `kept.json` — verdict가 keep인 문제만
4. `review.json` — 사람 확인 필요 큐
5. `summary.md` — 전체 통계 (유지/제외/검토 개수, 1층 규칙별 적중 수 등)

## 구현 메모
- 1층은 Python 순수 로직(정규식·집합 연산). 외부 API 불필요.
- 글자수는 `len(choice)`로 공백 포함 측정. 선지가 영어이므로 단어 수 대신 글자수 사용(사용자 확정 기준: 100자/3개).
- **고위험 교집합**: "글자수 신호(e)"가 "도입구 중복(a)" 또는 "토큰 중복률(c)"과 동시에 걸리면 결함 가능성이 높음 → 2층 우선 처리 큐로. (예: 미각 문제 = 길다 + 도입구 동일)
- 2층 LLM 호출은 배치 처리 + 재시도 + JSON 파싱 실패 시 재요청.
- 공식 정답을 **반드시 입력으로 고정**할 것. (LLM이 스스로 풀어 오답을 정답으로 착각하면 판정 전체가 틀어짐)
- 비용 제어: 기본은 1층 통과분만 2층 호출. `--full-llm` 옵션으로 전수 검사 가능.
- 결정타 문장은 반드시 **원문 그대로 인용**하도록 강제하고, 인용문이 실제 지문에 substring으로 존재하는지 코드로 검증(환각 방지).
