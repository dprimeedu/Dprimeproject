# 프라임에듀 자동화 EXE — 추가 기능 명세

> 작성일 2026-06-04. 그동안 수동으로 처리한 작업(서술형 추출 → 중요영작 O 표시 → 홈페이지 영작훈련 업로드/배정 → 고등 문장 자동분할)을 EXE에 내장하기 위한 기능 정리.

---

## 0. 배경 / 목적

현재 흐름(수동):
1. EXE가 학교별 **내신 통합본 xlsx**(`…1학기 기말고사 통합.xlsx`)와 EBS **부교재 통합본**(`2026년 영어영역 통합.xlsx`, `2026년 영어독해 통합.xlsx`)을 생성 — *이미 EXE가 함*.
2. (수동) **직전보강 HWP**에서 "서술형 대비 핵심 문장"을 추출해, 통합본 **중요영작 시트 F열**에 `O` 표시.
3. (수동) 중요영작 중 시험범위/`O` 대상만 추려 **홈페이지 영작훈련 단원**으로 만들고 **학생에게 배정** + AI 단어뜻 생성.
4. (수동/일부 자동) 고등학교 문장은 **20단어 초과 시 10~15단어로 분할**.

목표: 위 2·3·4를 EXE(또는 EXE↔홈페이지 API) 기능으로 자동화.

---

## 1. 기능 A — 직전보강 HWP → 서술형 추출 → 중요영작 `O` 표시

### 입력
- 직전보강 HWP: `통합본_…_직전보강_아잉카.hwp` (영어영역/영어독해 등 과목별)
- 대상 통합본 xlsx (그 과목)

### 처리
1. **HWP 표 파싱**: 문서 안의 표 중 머리글이 **"서술형 대비 핵심 문장"** 인 표(TableControl)를 모두 찾는다.
   - 각 데이터 행 = `[출처(수능/01/02…), 번호, 문장셀]`. 문장셀은 **영어 문단 + 한글 문단**.
   - 파서: `pyhwp`(`hwp5proc xml`로 덤프 후 `TableControl>TableRow>TableCell>…>Text` 파싱) 또는 한/COM. ※ 한 줄에 표가 빈 `<표>`로만 나오는 텍스트 추출(hwp5txt)은 부적합 → **XML 모델 파싱 필수**.
   - 영어/한글 구분: 셀 안 문단 중 **한글 포함=한글, 아니면 영어**.
2. **중요영작 시트 매칭**: 통합본의 `중요영작` 시트에서 **영어 열**을 찾아(아래 ⚠ 열 위치 가변), 추출한 서술형 영어와 비교.
   - 매칭 키 = **영문/숫자만 남기고 소문자화**(따옴표·공백·구두점 차이 무시).
   - **한 글자 오타/인코딩 깨짐은 동일 처리**: 정확 일치 외에 **유사도(SequenceMatcher) ≥ 0.95** 도 같은 문장으로 인정.
   - **제외**: 안내문/도표/의미가 다른 문장(유사도 < 0.95)은 표시하지 않음.
3. 매칭된 행의 **F열("중요영작")에 `"O"`** 기입.

### ⚠ 주의 (실측)
- **중요영작 시트 영어 열이 파일마다 다름**: 영어영역=**C열(영어)/B열(한글)**, 영어독해=**B열(영어)/C열(한글)**, 백현고2=B영어, 백현고1=C영어, 동백고2=C영어 → **열 고정 금지, 한글 포함 여부로 영어/한글 열 자동 감지**.
- 저장 시 **다른 시트 손상 금지**: openpyxl 통째 저장은 수식 시트(예 `문제출력`)의 **캐시값을 떨어뜨림** → 백업 후 **해당 시트 XML만 패치**(zip 내 `xl/worksheets/sheetN.xml`만 교체)하는 방식 권장. 최소한 저장 전 자동 백업.

## 2. 기능 B — 고등학교 영작 문장 자동 분할 (10~15단어)

> ※ 홈페이지(웹) 업로드 경로에는 **이미 구현·배포됨**(2026-06-04). EXE가 홈페이지 업로드/Import API를 쓰면 자동 적용됨. EXE가 자체 분할까지 하려면 아래 동일 로직 사용.

### 규칙
- **고등학교(고1/고2/고3)만** 적용. 중학교/초등/기타는 **그대로**.
- 한 문장이 **20단어 초과**면 분할:
  - 끊는 기준: **주절/종속절 경계, 쉼표, 대시(— -), 관계대명사(who/which/that/where…), 등위접속사(and/but/or) 앞**.
  - 각 조각 **10~15단어** 목표(인접 절 병합, 한 조각 20단어 넘기지 않기).
- **영어는 원문 단어를 글자까지 그대로 보존**(영작 채점이 단어 일치라 변형 금지). 한글은 조각별로 정렬.
- 각 조각이 1문제가 됨(긴 1문장 → 여러 짧은 문제).

### 구현 메모
- 홈페이지 함수: `writing/services/ai.py::split_sentence_clauses(english, korean)` (Gemini로 끊는 위치+한글 산출 → **원문 단어를 그 개수만큼 잘라** 영어 보존 → 10~15 병합 → 영어 보존 검증, 실패 시 미분할).
- 단어뜻 힌트(word_hints)는 분할 **후** 단어 위치대로 잘라 쓰거나 재생성.

## 3. 기능 C — 중요영작 → 홈페이지 영작훈련 업로드 + 학생 자동 배정

### 입력
- 업로드할 (한글, 영어) 문장 목록 + **단원 제목/학년/출판사** + **배정 학생 목록**.
- 출처 두 가지:
  - **학교 내신 통합본**(예 `백현고2 1학기 기말고사 통합.xlsx`): `중요영작` 시트 **전체**가 그 학교 기말 범위 → 전부 업로드.
  - **EBS 부교재 통합본**(`영어영역/영어독해 통합.xlsx`): 큰 책 → **학교별 시험범위(강-번호 목록)로 필터** + 기능 A의 `O` 표시된 것 위주.

### 처리(홈페이지에 생성)
- `WritingUnit`(title, publisher, grade) 1개 생성.
- 각 문장 → `WritingProblem`(index, korean=한글, english=영어, word_hints=[]).
- **고등이면 기능 B 분할 적용** → 조각 단위 문제.
- **AI 단어뜻(word_hints) 생성**(Gemini batch).
- 학교 학생 전원에게 `UnitAssignment` 생성(학교학년으로 매칭, login_id=`primeedu`+N).

### 권장: 홈페이지에 **영작 Import API 신설**
- 기존 단어(vocab)용 토큰 API와 동일 패턴 권장: 헤더 `X-Vocab-Token`(또는 신규 토큰), `VOCAB_IMPORT_TOKEN` 사용.
- 예: `POST /training/writing/api/import/`
  ```json
  {
    "token": "...",
    "units": [{
      "title": "백현고2 1학기 기말 중요영작",
      "grade": "고2", "publisher": "2026 내신 통합",
      "problems": [{"korean": "...", "english": "..."}, ...],
      "assign_login_ids": ["primeedu19","primeedu20", ...],
      "assign_school": "백현고2"        // 또는 학교학년으로 자동 매칭
    }]
  }
  ```
- 홈페이지 측 동작: 단원 생성 → (고등 자동분할) → 문제 생성 → 힌트 생성(백그라운드) → 배정. (현 `upload_view`/`_start_hint_generation(split_first=True)` 로직 재사용.)
- ※ 현재는 **브라우저 엑셀 업로드(`/training/writing/admin/upload/`)만** 존재. EXE 자동화를 위해 토큰 API 추가가 필요.

### 학생 매칭 참고(실측)
| 학교 | 학생(login_id) |
|---|---|
| 동백고2 | primeedu8~15 (김가현·나윤희·박서현·이다영·이상윤·정지유·조성하·최혜련) |
| 백현고1 | primeedu17(정예원)·primeedu18(허지윤) |
| 백현고2 | primeedu19~23 (권성연·문준혁·송지우·원송현·장시은) |
| 초당고3 | primeedu33(김민세) — 2026-06-04 수동 완료(영어영역 부교재, 시험범위 18~29강) |

## 4. 기능 D — D열(단원) 표기 양식 통일

- 과목/파일마다 단원 표기가 다름: `1 No. 1`(영어독해) vs `1-1`/`1-G`/`T14-1`(영어영역) vs `9-1`/`3-3`(학교 내신).
- **표준 형식 `{강}-{번호}` 로 통일** 권장: `" No. "` → `"-"` 치환 (`1 No. 6` → `1-6`, `T1 No. 5` → `T1-5`).
- 셀 서식(`@` 텍스트·폰트)은 유지.

## 5. 공통 데이터/형식 참고

### 통합본 `중요영작` 시트 컬럼
| 열 | 헤더 | 내용 |
|---|---|---|
| A | 색인 | 번호 |
| B | 영작 | (파일에 따라) 한글 또는 영어 |
| C | 해석 | (파일에 따라) 영어 또는 한글 |
| D | 단원 | 강-번호 (형식 가변 → 기능 D로 통일) |
| E | 회화/Extra/기타여부 | 부가 |
| F | 중요영작 | **기능 A가 `O` 기입** |

→ **B/C 중 영어 열은 한글 포함 여부로 자동 판별**(고정 금지).

### 홈페이지 영작 데이터 모델
- `WritingUnit`(title, publisher, grade[초1~고3/기타])
- `WritingProblem`(unit, index, **korean**(한글 힌트), **english**(영어 정답), word_hints=`[{"word","meaning","proper_noun"}]`)
- `UnitAssignment`(student, unit, assigned_by)
- 학생 = `member.Member`(login_id=`primeedu`+N, username=이름, 비번 123456, is_approved=True)

### 매칭 규칙(기능 A·C 공통)
- 정규화: `re.sub(r'[^a-z0-9]','', text.lower())`
- 동일 판정: 정규화 일치 **또는** `difflib.SequenceMatcher` 비율 ≥ 0.95(한 글자 오타·따옴표 깨짐 흡수).
- 제외: 안내문·도표·의미 상이 문장.

## 6. 권장 구현 순서

1. **기능 A**(HWP 서술형 → 중요영작 O) — EXE 단독 가능(엑셀만 수정). 우선순위 높음.
2. **기능 D**(D열 양식 통일) — EXE 단독, 간단. A와 함께.
3. **홈페이지 영작 Import API 신설**(기능 C 토대) — 웹 측 작업.
4. **기능 C**(중요영작 → 영작훈련 업로드/배정) — EXE가 API 호출. 고등 자동분할(B)은 홈페이지가 처리.
5. (선택) EXE 자체 분할(B) — 홈페이지 API를 못 쓸 때만.

## 7. 비고 / 리스크
- HWP 파싱은 환경 의존(한/COM은 보안팝업, pyhwp는 순수파싱이라 EXE 배포에 적합).
- 엑셀 저장 시 **수식 시트 캐시 손상** 주의(기능 A ⚠ 참조) — 백업 필수.
- AI(Gemini) 호출은 **GEMINI_API_KEY** 필요(홈페이지 .env에 존재). EXE에서 직접 호출 시 키 관리 필요 → **분할·힌트는 홈페이지에 위임** 권장.
- 시험범위(EBS 부교재용 강-번호)는 학교별로 다름 → EXE 입력(또는 학생관리자료 시트)에서 받아야 함. 학교 내신 통합본은 범위 필터 불필요(전체가 기말 범위).
