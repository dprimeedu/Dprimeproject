## 🅑 이번 주 토요일 — 협업자 통보 사항

### A. GitHub PR 만들기

[PR_본문_복붙용.md](PR_본문_복붙용.md) (개인 로컬 파일, gitignored) 의 내용 그대로:

2. 제목/본문 복붙
3. Create pull request

### B. 협업자에게 추가 통보 (DB 통일 + Sync 시스템)

[PR_본문_복붙용.md](PR_본문_복붙용.md) 아래쪽 섹션 "📨 협업자에게 추가 전달 내용" 그대로 카톡/메시지 전송. 핵심:

1. **DB 통일**: Postgres → SQLite (협업자가 이미 SQLite라 변동 없음)
2. **마이그레이션 신규 적용 필요**: `git pull` → `python manage.py migrate`
3. **신규 sync 시스템**: `python manage.py sync_exam_data` — 모의고사 엑셀 → DB 자동 적재
4. **NAS는 이미 재적재 완료**: 운영 사이트 정상 동작 중
5. **보안 권장**: 강한 비번으로 교체 협의 (지금 `1`)

## 🅒 보류 (결정 필요한 것)

### C-1. Django admin "지금 sync 실행" 버튼 (1-2시간)
웹UI에서 클릭으로 sync 트리거. 백그라운드 처리 필요 (현재 동기 1-2분 소요).

### C-2. 자동 스케줄러 (0.5일)
Synology Task Scheduler가 5분/15분/1시간 간격으로 sync 실행. mtime 체크로 변경 없으면 skip.

### C-3. 모의고사 데이터 HWP 다운로드 (4-7일 예상)
사용자 제안 아키텍처:
- 서버: 데이터 → AES 암호화 → `.pedu` 파일 응답
- 클라이언트: PyInstaller EXE, `.pedu` 복호화 + 한컴 COM으로 HWP 생성
- 이미 본인 보유 코드(`Z:\home\Drive\문서 자동화 작업\완성(코드)\B. 부교재 출력.py`)에 자동화 로직 있음 — 재활용

상세 설계는 별도 작업으로.

### C-4. 마이그레이션 0001/0002 의존성 정리
`KeyTable`이 0002에서 만들어지는데 detail 테이블들이 0001에서 KeyTable에 FK를 걸어서 lazy reference 에러. 지금은 우회로 동작 중. 다음 마이그레이션 작업 때 정리하면 됨.

자세한 건 [docs/history/2026-05-18_DB통일_sync시스템.md](docs/history/2026-05-18_DB통일_sync시스템.md#5-발견한-이슈--향후-정리할-것) 5-1 항목 참조.

### C-5. 운영 NAS 백업 파일 정리
`/volume1/Docker/Dprimeproject/db.sqlite3.backup-20260518-214951` (139MB) — 1주일 정도 두고 문제 없으면 삭제.
